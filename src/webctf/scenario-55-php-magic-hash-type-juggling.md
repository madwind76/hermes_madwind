# PHP Loose Comparison and Magic Hash Authentication Bypass — Web CTF Scenario

## 1. 개요 및 스토리
- **문제명**: Hash Validator (해시 보안 검증기)
- **난이도**: Medium
- **핵심 컨셉**: PHP 프로그래밍 언어의 동적 타입 캐스팅 허점과 해시 문자열 비교의 논리적 맹점을 파고드는 **PHP Type Juggling 및 Magic Hash** 취약점 문제입니다. 대상 애플리케이션은 관리자 세션 복구를 위해 사용자에게 임시 핀(PIN) 코드 해시를 대조 검증하는 기능을 제공합니다. 그러나 해시 결과와 사용자 제출 값을 매핑 비교하는 과정에서, 엄격한 일치 비교 연산자(`===`) 대신 약한 비교 연산자(`==`)를 사용합니다. 공격자는 생성된 해시 결과가 과학적 표기법 부동 소수점(예: `0e123...`)으로 해석될 수 있는 특정 문자열 구조를 가질 때, 비교 값을 단순한 정수 0 혹은 유사한 매직 해시 값으로 입력하여 패스워드나 인증 토큰을 전혀 알지 못하더라도 인증을 통과해 관리자 계정을 확보합니다.

## 2. 문제 설계 및 구조 (Architecture)

### 2.1 구성 요소
- **Reset Token Verifier (`reset.php`)**:
  - 관리자가 입력한 복구 토큰을 백엔드에서 받아서 DB에 저장된 해시된 값(`md5` 또는 `sha256`)과 비교 수행.
  - 비교 구문 코드 사양:
    `if ($input_token == $db_stored_hash) { ... }`
- **Flag 위치**:
  - `reset.php`에서 인증 비교를 우회한 뒤 반환되는 세션 권한을 획득하고, 관리자 전용 정보 페이지 `/admin_panel.php`를 호출했을 때 출력되는 텍스트 본문 내 플래그.

### 2.2 취약점 지점
1. **Loose Comparison Operator (`==`)**:
   - PHP에서 `==` 연산자는 양쪽 항의 데이터 타입이 다르면 임의로 타입을 변환하여 동등성을 검증하려 합니다.
2. **Magic Hash (0e... pattern)**:
   - 만약 문자열이 `0e`로 시작하고 그 뒤가 전부 숫자로 이루어져 있다면(예: `0e5482390...`), PHP는 이를 부동 소수점 지수 표기법(`0 * 10^5482390`)으로 판단하여 연산 결과값인 **`0`** (Float)으로 해석해 버립니다.
   - 데이터베이스에 저장된 마스터 암호나 임시 토큰의 해시값 중 우연히 `0e...` 패턴을 가지는 해시가 매칭되고, 공격자가 입력한 값의 해시 또한 `0e...` 패턴을 띠면, PHP는 `0 == 0`으로 수긍하여 인증을 통과시켜 줍니다.

## 4. 상세 풀이 흐름 (Exploitation Flow)

### Step 1. PHP 소스코드 유출 및 취약 연산자 확인
1. 공격자는 웹 루트 상의 백업 파일이나 깃 이력을 탐색하여 `reset.php` 소스코드를 확보하거나, 에러 디버그 메시지를 보고 PHP 환경임을 확인합니다.
2. 토큰을 대조하는 구문이 아래와 같이 구현되어 있음을 식별합니다:
   ```php
   $secret_token = "0e542981329048123908412390841230"; // 예시용 하드코딩 혹은 생성된 매직 해시
   if ($_POST['reset_token'] == $secret_token) {
       $_SESSION['role'] = 'admin';
   }
   ```

### Step 2. Magic Hash 조건 성립 패턴 확인
서버의 `$secret_token` 값이 `"0e"`로 시작하는 MD5 해시 문자열이라고 가정합니다.
- MD5 해시 결과가 `0e`로 시작하며 뒤이어 숫자만 나오는 대표적인 매직 해시 원본 단어들을 리포트에서 검색합니다:
  - `"240610708"` -> MD5: `0e462097431906509449523513806310`
  - `"QNKCDZO"` -> MD5: `0e830400451993494058024219903391`
  - `"aabg7XSs"` -> MD5: `0e087386482136013740957780965295`
- 공격자가 `reset_token` 값으로 `"QNKCDZO"`를 전송하면:
  1. 서버는 입력받은 값의 해시인 `0e830400...` 과 저장된 비밀 값 `0e542981...`을 비교합니다.
  2. 코드 내부: `if ("0e830400..." == "0e542981...")`
  3. PHP 파서는 양측을 부동 소수점으로 변환하여 `0 == 0`으로 평가합니다.
  4. 두 문자열이 실제로는 다름에도 불구하고 조건문은 **참(True)**을 리턴합니다.

### Step 3. 우회 입력값 전송
1. 공격자는 취약한 토큰 검증 페이지에 매직 해시를 발생시키는 단어를 인자로 기입해 POST 요청을 보냅니다.
   - **요청 데이터**:
     `reset_token=QNKCDZO` (혹은 서버에서 해시하지 않고 문자열 비교만 직접 수행할 경우, 값 자체를 `0` 또는 `0e88888` 형태로 주입)
2. 서버는 느슨한 자동 형변환 비교 메커니즘을 작동시켜 세션 인증을 정상 수락합니다.

### Step 4. flag 획득
1. 성공적으로 획득된 세션 쿠키를 지니고 `/admin_panel.php` 페이지로 리다이렉트합니다.
2. 권한 통과와 함께 렌더링된 메인 화면에서 플래그(`FLAG{php_loose_comparison_magic_hash_juggling_bypass}`)를 획득합니다.

## 5. 취약점 유발 백엔드 코드 스니펫 (PHP)

```php
<?php
// reset.php (취약한 PHP 비교 루틴 예시)
session_start();

// 서버에 생성되어 보관 중인 마스터 관리자 패스워드 복구 키 (우연히 매직해시 구조를 지님)
// 실제 DB에서 불러온 비밀키 데이터가 sha1("비밀키") 형태로 저장되어 있다고 가정
$db_stored_hash = "0e830400451993494058024219903391"; 

if (isset($_POST['reset_token'])) {
    $user_input = $_POST['reset_token'];
    
    // 취약점 지점: 사용자가 입력한 문자열과 DB 해시값을 느슨한 비교(==)로 검증
    // user_input의 해시값(혹은 입력값 자체)이 0e[0-9]+ 규격을 지니면 0 == 0으로 참 처리됨
    // 예: user_input으로 "240610708"을 넣으면 md5 해시 결과가 0e46209...이 되어 통과
    $input_hash = md5($user_input);

    if ($input_hash == $db_stored_hash) {
        $_SESSION['role'] = 'admin';
        header("Location: admin_panel.php");
        exit();
    } else {
        echo "<h4>인증 코드가 일치하지 않습니다.</h4>";
    }
}
?>
```

## 6. 방어 및 완화 기법 (Mitigation)

1. **엄격한 비교 연산자 강제 (Strict Comparison)**:
   - 데이터 타입까지 정밀하게 비교하여 자동 형변환의 개입을 원천적으로 막는 엄격한 비교 연산자(`===` 또는 `!==`)를 상시 사용합니다.
     ```php
     if ($input_hash === $db_stored_hash) { ... }
     ```
2. **해시 보안 대조 함수 연동 (Use hash_equals)**:
   - 타이밍 공격(Timing Attack) 방어와 엄격한 이진 대조를 동시에 제공하는 PHP 기본 내장 함수 `hash_equals()`를 연동하여 중요 값들을 검증합니다.
     ```php
     if (hash_equals($db_stored_hash, $input_hash)) { ... }
     ```
3. **타입 캐스팅 명시**:
   - 비교 수행 직전, 데이터의 변수 유형을 문자열(`(string)`)로 강제 강압 캐스팅하여 숫자로 오해되는 컴파일러 파싱 흐름을 방지합니다.
4. **PHP 최신 버전 가동**:
   - PHP 8.0 이상 버전부터는 문자열과 정수 간 비교 시 문자열이 숫자로 판별되지 않으면 숫자를 문자열로 변환해 비교하도록 규칙이 개선되어 일부 취약점이 방어되나, 해시 대 해시 문자열 비교에서는 여전히 `===` 연산자를 사용하는 것이 안전 표준입니다.

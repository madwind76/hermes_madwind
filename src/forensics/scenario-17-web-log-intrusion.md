# 교묘한 웹서버 공격 추적 (Web Log Intrusion Analysis)

> **난이도**: 초중급  
> **소요 시간**: 25~30분  
> **참고 picoCTF 문제**: 웹 기반 침해 사고 조사 및 타임라인 분석 (Apache/Nginx Access Log 포렌식)

## 1. 배경 시나리오
사내에서 웹 서비스를 제공 중인 리눅스 기반 웹 서버에서 비정상적인 CPU 사용량 폭증과 함께 시스템 파일이 무단으로 유출되는 침해 사고가 발생했습니다. 보안 관제 부서는 당일 작동한 웹 서버의 웹 액세스 로그 파일인 `access.log`를 추출하여 분석가에게 전달했습니다. 분석가는 해킹 침투가 시작된 공격자의 외부 IP 주소, 업로드 및 실행된 악성 웹셸(Webshell)의 파일명, 그리고 해당 웹셸이 첫 요청될 때 반환된 HTTP 상태 코드를 조사해야 합니다.

## 2. 제공 파일
* `access.log` (Nginx/Apache 공통 로그 포맷의 웹 서버 액세스 기록 파일, 약 5,000행)

## 3. 문제 목표
텍스트 기반 웹 서버 로그 파일의 구조(IP, 시각, HTTP 메서드, URI, 상태 코드, User-Agent)를 이해하고, `grep`, `awk`, `cut` 등의 리눅스 텍스트 필터링 명령어 조합을 사용하여 대량의 정상 노이즈 속에서 공격 시도의 타임라인을 파악하여 특정 침해 정보(공격자 IP, 웹셸 파일명, 응답 코드)를 확보합니다.

## 4. 의도한 풀이 흐름
1. **로그 포맷 및 패턴 식별**:
   * 제공된 `access.log` 파일을 확인하여 표준 결합 로그 포맷(Combined Log Format)을 따르고 있는지 관찰합니다.
2. **공격 시도 탐색 (취약점 스캔 검출)**:
   * 해커들이 흔히 사용하는 취약점 검색 시도 패턴(예: `../`, `etc/passwd`, `union select`, `select * from` 등)을 로그 전체에서 검색합니다:
     ```bash
     grep -E "\.\./|union|select" access.log
     ```
   * 검색 결과 특정 IP(예: `192.168.10.45`)가 다량의 에러 코드(400, 404)를 띄우며 다양한 인젝션 및 탐색 경로 공격을 수행했음을 파악합니다.
3. **웹셸 업로드 유추**:
   * 공격자 IP `192.168.10.45`가 시도한 요청 중 웹 애플리케이션의 업로드 엔드포인트(예: `/upload.php`, `/file_upload` 등)로의 POST 요청과 성공 코드(200 OK)를 필터링합니다:
     ```bash
     grep "192.168.10.45" access.log | grep "POST"
     ```
4. **웹셸 실행 확인**:
   * 파일 업로드 직후, 업로드 디렉터리 경로(예: `/uploads/`, `/files/`, `/data/`)에 새로 생성되어 외부에서 호출된 PHP 또는 JSP 등 웹셸 스크립트 파일을 찾아냅니다.
     (예: `GET /uploads/backdoor.php?cmd=whoami HTTP/1.1" 200`)
   * 해당 웹셸 파일명(`backdoor.php`)과 처음 실행 시 반환된 HTTP 응답 코드(`200`)를 식별합니다.
5. **플래그 조립**:
   * 찾은 세 가지 요소인 **공격자 IP, 웹셸 파일명, 첫 응답 코드**를 언더스코어로 결합해 플래그를 생성합니다:
     `picoCTF{192.168.10.45_backdoor.php_200}`

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<attacker_ip>_<webshell_name>_<http_status_code>}`
* **예시**: `picoCTF{192.168.10.45_backdoor.php_200}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 약 4,000~5,000행의 정상 사용자 접속 기록 로그(정상 페이지 GET 요청, 이미지/CSS 로딩, 상태 코드 200/304 위주)를 생성하거나 수집합니다.
  2. 중간 시점에 공격자 IP(`192.168.10.45`)의 웹 스캔 및 공격 페이로드 로그를 100여 개 삽입합니다.
  3. 스캔 기록 바로 뒤에 파일 업로드 취약점을 타깃으로 하는 POST `/upload.php` 로그(성공 코드 200)를 배치합니다.
  4. 업로드된 경로 `/uploads/backdoor.php`로 들어오는 파라미터 기반 웹셸 명령 제어 요청 로그를 약 10건 추가합니다:
     `192.168.10.45 - - [28/Jun/2026:05:14:22 +0900] "GET /uploads/backdoor.php?cmd=cat%20/etc/passwd HTTP/1.1" 200 842 "-" "Mozilla/5.0 ..."`
  5. 최종 `access.log` 파일을 텍스트 형식으로 정제해 출제 파일로 지정합니다.
* **출제 포인트**: 
  * 침해 사고 대처(Incident Response) 및 보안 로그 감사 시 필수적으로 요구되는 대규모 로그 텍스트 분석 필터링 기술과 공격 시나리오(스캐닝 -> 익스플로잇 -> 웹셸 설치 -> 명령 수행) 타임라인 추적 역량을 평가합니다.

## 7. 트러블슈팅 및 힌트
* **Q. 로그 행 수가 너무 많아서 스크롤만으로는 시점을 잡기 어렵습니다.**
  * A. 통계 쿼리를 작성하여 비정상적으로 많이 호출된 파일이나 특이한 IP를 먼저 분류하십시오:
     `cat access.log | awk '{print $1}' | sort | uniq -c | sort -nr` (접속 IP 빈도 통계)
     `cat access.log | cut -d'"' -f2 | awk '{print $2}' | sort | uniq -c | sort -nr | head -n 30` (요청 주소 빈도 통계)
* **Q. backdoor.php 말고도 정상적인 php 파일(index.php, upload.php)이 다수 나옵니다.**
  * A. 공격자가 새로 생성해서 요청을 보낸 파일은 정상 사용자의 호출 기록 분포에 비해 접속 IP 수가 공격자 IP 1개로만 국한되어 있고 시각이 국지적으로 몰려 있는 등의 이상 징후(Anomaly)를 보입니다.

## 8. 학습 포인트
* **웹 아키텍처 로그 분석**: 아파치/엔직스 등의 대중적 웹 서버가 I/O 활동을 텍스트 파일에 직렬화하여 기록하는 메커니즘을 상세히 체득합니다.
* **침해 사고 조사(IR) 타임라인**: 유입 경로 식별부터 행위 분석까지 유기적으로 연결되는 해킹 공격자의 침투 킬체인(Kill Chain) 분석 관점을 훈련합니다.

# 거대한 모래사장 속 바늘 (Needle in the Text Stack)

> **난이도**: 초급  
> **소요 시간**: 15~20분  
> **참고 picoCTF 문제**: `Lookey here` (picoCTF 2022), `Verify` (picoCTF 2024)

## 1. 배경 시나리오
사내 파일 서버에 악성 백도어가 설치되었던 흔적을 발견했습니다. 용의자는 추적을 방해하기 위해 수백 개의 중첩된 폴더 구조를 생성하고, 각각의 폴더 내부에 의미 없는 랜덤 문자열과 무작위 해시값을 담은 텍스트 파일 1,000여 개를 분산해 두었습니다. 다행히 침입 탐지 로그 시스템에서 실시간으로 기록한 **악성 행위가 발생한 최종 페이로드 파일의 SHA-256 체크섬(Checksum)** 1건을 확보해 둔 상태입니다. 분석가는 이 체크섬과 일치하는 진짜 증거 파일이 무엇인지 탐색해 그 내부의 플래그를 찾아내야 합니다.

## 2. 제공 파일
* `checksum.txt` (공격 파일의 SHA-256 해시값 문자열이 적힌 텍스트 파일)
* `files/` (수십 개의 중첩 폴더와 1,000여 개의 난독화 텍스트 파일들이 저장된 디렉터리)

## 3. 문제 목표
많은 수의 파일이 존재하는 복잡한 디렉터리 트리 구조 내에서 리눅스 셸 스크립트 또는 파이썬 스크립트를 활용해 고속으로 전체 파일의 SHA-256 체크섬을 전수 조사하고, 대조 필터링을 거쳐 일치하는 유일한 파일을 식별하여 기밀 플래그를 찾아냅니다.

## 4. 의도한 풀이 흐름
1. **타깃 해시 파악**:
   * `checksum.txt` 파일을 확인하여 우리가 찾아야 할 대상 파일의 SHA-256 값을 확인합니다.
     (예: `687f87df607d7cc017d23d8c1c4e7fb2610d...`)
2. **리눅스 셸 명령어를 이용한 고속 해시 전수검사**:
   * `find` 명령어로 `files/` 하위의 모든 파일 경로를 재귀적으로 찾습니다.
   * `xargs` 또는 `while loop`을 이용해 각 파일의 SHA-256 해시를 일괄 계산하고 대조합니다:
     ```bash
     find files/ -type f -print0 | xargs -0 sha256sum | grep $(cat checksum.txt)
     ```
   * 또는 1줄 스크립트를 실행합니다:
     ```bash
     find files/ -type f -exec sha256sum {} + | grep -f checksum.txt
     ```
3. **일치 파일 식별**:
   * 필터링된 결과로 출력된 파일 경로(예: `files/subfolder_c/part_82/random_412.txt`)를 식별합니다.
4. **플래그 확인**:
   * 해당 경로의 텍스트 파일을 읽어 플래그를 획득합니다.
     `cat files/subfolder_c/part_82/random_412.txt`
     (출력 결과 예: `picoCTF{f1nd_th3_n33dl3_1n_th3_st4ck}`)

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<flag_string>}`
* **예시**: `picoCTF{f1nd_th3_n33dl3_1n_th3_st4ck}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 파이썬 스크립트를 이용해 무작위 중첩 디렉터리를 만들고 랜덤 텍스트 파일을 1,000여 개 분산 생성합니다.
  2. 그중 특정 폴더의 파일 하나에 실제 플래그 `picoCTF{f1nd_th3_n33dl3_1n_th3_st4ck}`를 삽입합니다.
  3. 플래그가 들어간 파일의 SHA-256 값을 계산해 `checksum.txt`로 내보냅니다:
     `sha256sum files/path/to/flag_file.txt | awk '{print $1}' > checksum.txt`
  4. 모든 파일의 크기와 확장자를 동일하게 구성하여 파일 탐색기 정렬만으로는 특정 파일이 튀어 보이지 않게 위장 처리(Stealth)합니다.
* **출제 포인트**: 
  * 파일이 대량으로 흩어진 침해 사고 대응 상황(DFIR)에서 정규표현식, find, xargs, sha256sum 등의 터미널 파이프라인 연동 기술을 능숙하게 활용하여 증거 탐색 효율성을 극대화하는 훈련을 합니다.

## 7. 트러블슈팅 및 힌트
* **Q. xargs로 해시를 계산하려는데 파일명 사이에 공백이나 특수 문자가 있어서 에러가 납니다.**
  * A. `find` 명령어 사용 시 `-print0` 옵션을 주고, `xargs` 측에는 `-0` 옵션을 지정해 파일명 구분을 개행이 아닌 NULL 바이트 기준으로 하도록 정규화하면 공백이 섞여 있는 파일명도 누수 없이 정상 계산할 수 있습니다.
* **Q. 단순 grep으로 전체 폴더에서 picoCTF{ 문자열을 찾으면 더 빨리 풀 수 있지 않나요?**
  * A. 네, 본 문제에서는 단순 grep으로 가능할 수 있으나, 출제자가 플래그 텍스트 자체도 난독화하거나 암호화된 파일 상태로 배치한 경우에는 체크섬 매칭이 유일한 해결 수단이 됩니다. 해시 비교 기법을 반드시 숙지해 두십시오.

## 8. 학습 포인트
* **체크섬 무결성 검증**: 해시 함수를 통해 데이터의 동일성(무결성)을 고속 증명하는 방식과 포렌식 실무에서의 해시 매칭 분석(Hash Matching) 기법을 학습합니다.
* **리눅스 셸 명령어 체인**: `find`, `xargs`, `sha256sum`, `grep`을 조합하여 대규모 비구조화 파일 군 속에서 유의미한 단서를 초 단위 속도로 색인해 내는 셸 마스터링 기술을 습득합니다.

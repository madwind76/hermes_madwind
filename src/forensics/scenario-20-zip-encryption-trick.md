# ZIP 헤더 패스워드 트릭 (ZIP Encryption Pseudo-Lock)

> **난이도**: 초중급  
> **소요 시간**: 20~25분  
> **참고 picoCTF 문제**: ZIP 아카이브 헤더 구조 우회 (가짜 암호화 복구 기법)

## 1. 배경 시나리오
보안 연구소의 백업 드라이브에서 기밀 정보가 백업된 `pseudo_locked.zip` 파일이 발견되었습니다. 이 파일의 압축 해제를 시도하면 표준 압축 프로그램(WinZip, unzip 등)이 비밀번호 입력을 요구합니다. 하지만 포렌식 분석 도구로 무차별 대입(Brute Force) 공격을 아무리 수행해도 유효한 패스워드가 매칭되지 않습니다. 분석 결과, 피의자는 압축 비밀번호를 실제로 설정한 것이 아니라, **ZIP 파일 헤더의 암호화 설정 지시 비트(General Purpose Bit Flag)**를 인위적으로 조작하여 패스워드 입력 모드를 강제 활성화해 둔 것으로 파악되었습니다. 

## 2. 제공 파일
* `pseudo_locked.zip` (압축 해제 시 비밀번호를 요구하는 위조된 ZIP 아카이브 파일)

## 3. 문제 목표
ZIP 아카이브 파일 포맷 구조(Local File Header 및 Central Directory Header)를 이해하고, 헥스 에디터를 사용하여 암호화 여부를 결정하는 범용 플래그 비트(General Purpose Bit Flag)의 오프셋 위치를 찾아내어 이를 `0`으로 정상화함으로써 비밀번호 입력 없이 아카이브 내부 파일을 안전하게 무력화 추출합니다.

## 4. 의도한 풀이 흐름
1. **오류 재현**:
   * 터미널에서 `unzip pseudo_locked.zip`을 실행하여 실제로 패스워드 입력을 요구하는지 확인합니다.
2. **바이너리 구조 분석 (Hex Editor 사용)**:
   * 헥스 에디터로 `pseudo_locked.zip` 파일을 로드합니다.
   * **로컬 파일 헤더 (Local File Header)** 영역을 찾습니다:
     * 시그니처: `50 4B 03 04` (PK\x03\x04)
     * 시그니처 기준 오프셋 6번 바이트 위치에 존재하는 범용 비트 플래그(General Purpose Bit Flag, 2바이트)를 식별합니다.
     * 현재 값이 `01 00` (리틀 엔디언 구조 상 첫 번째 비트인 비트 0이 1로 설정됨 -> 암호화 활성화 상태)으로 적혀 있음을 파악합니다.
   * **중앙 디렉터리 헤더 (Central Directory Header)** 영역을 찾습니다:
     * 시그니처: `50 4B 01 02` (PK\x01\x02)
     * 시그니처 기준 오프셋 8번 바이트 위치에 존재하는 동일 범용 비트 플래그 값을 식별합니다.
     * 이 역시 `01 00`으로 조작되어 있음을 확인합니다.
3. **헤더 비트 수정**:
   * 로컬 파일 헤더의 오프셋 6번 플래그 값을 `00 00`으로 덮어씁니다.
   * 중앙 디렉터리 헤더의 오프셋 8번 플래그 값도 `00 00`으로 덮어씁니다.
     *(만약 압축 파일 내에 들어 있는 파일 엔트리가 여러 개라면, 모든 파일 엔트리의 로컬 헤더와 중앙 디렉터리 헤더의 플래그 비트를 모두 수정해야 합니다)*
4. **저장 및 해제 시도**:
   * 변경된 바이너리를 `unlocked.zip` 파일로 저장합니다.
   * `unzip unlocked.zip` 명령을 실행합니다. 이번에는 비밀번호를 전혀 묻지 않고 아카이브가 즉각 압축 해제됨을 확인합니다.
5. **플래그 확인**: 추출된 텍스트 파일을 열어 최종 기밀 플래그를 확인합니다.
   (최종 플래그: `picoCTF{zip_pseudo_encryption_bypassed_via_headers}`)

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<flag_string>}`
* **예시**: `picoCTF{zip_pseudo_encryption_bypassed_via_headers}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 텍스트 파일 `flag.txt`에 최종 플래그 값(`picoCTF{zip_pseudo_encryption_bypassed_via_headers}`)을 기록합니다.
  2. 비밀번호 없이 일반 압축을 수행합니다:
     `zip pseudo_locked.zip flag.txt`
  3. 생성된 `pseudo_locked.zip`을 헥스 에디터로 엽니다.
  4. 로컬 파일 헤더 시작부(`50 4B 03 04`) 기준 6번 오프셋의 `00 00`을 `01 00`으로 덮어씁니다.
  5. 파일 하단 부근에 있는 중앙 디렉터리 헤더 시작부(`50 4B 01 02`) 기준 8번 오프셋의 `00 00`을 `01 00`으로 덮어씁니다.
  6. 저장 후 일반 압축 도구로 풀 때 패스워드 입력 창이 뜨는지 자가 진단한 후 파일을 배포합니다.
* **출제 포인트**: 
  * 파일 포맷 헤더 내에 명시되는 다양한 기능 지시용 플래그 비트의 역할을 이해하고, 헥스 오프셋 수정 제어를 통해 이를 해제하는 안티 포렌식 무력화 기법을 학습시킵니다.

## 7. 트러블슈팅 및 힌트
* **Q. 플래그 비트를 바꿨는데도 여전히 압축 해제 시 비밀번호나 CRC 에러가 뜹니다.**
  * A. 로컬 파일 헤더(`50 4B 03 04`로 시작)의 플래그 비트와 중앙 디렉터리 헤더(`50 4B 01 02`로 시작)의 플래그 비트를 둘 다 수정했는지 확인하십시오. 한쪽만 수정하면 헤더 간 데이터 일관성 에러로 인해 파일이 깨지거나 여전히 패스워드 처리가 유지됩니다.
* **Q. ZIP의 일반적인 비밀번호 압축 방식과 이 트릭의 동작 차이는 무엇인가요?**
  * A. 실제 비밀번호 압축을 진행한 경우, 파일 헤더 플래그 수정 외에도 데이터 영역 자체가 비밀번호 키로 암호화(Crypto) 처리되어 있으므로 비트만 고쳐 해제하려 하면 본문 CRC 에러가 나며 깨진 파일이 출력됩니다. 반면 이 트릭은 물리 데이터 영역은 평문 상태를 유지하고 헤더만 조작했기 때문에 비트 복구 즉시 원본이 복원됩니다.

## 8. 학습 포인트
* **ZIP 아카이브 포맷 헤더 규격**: 파일 묶음 정보(Local Header)와 압축 파일 전체 목차 정보(Central Directory)가 복합 보존되는 구조를 습득합니다.
* **암호화 속성 우회**: 안티 포렌식 기법 중 하나인 의도적 데이터 속성 위조(Pseudo-encryption)를 식별하고, 저수준 바이트 패치를 통해 복제 데이터를 추출하는 능력을 기릅니다.

# 윈도우 가상 사설망 연결 흔적 프리페치 분석 (VPN Client Execution Timeline)

> **난이도**: 초급  
> **소요 시간**: 15~20분  
> **참고 picoCTF 문제**: 비인가 암호화 터널 가동 시각 규명 (Windows VPN Prefetch 분석)

## 1. 배경 시나리오
공격자가 외부 네트워크 감시 장비를 우회하여 사내 연구망의 핵심 서버에 은밀히 통신하기 위해, 비인가 가상 사설망(VPN) 클라이언트인 **OpenVPN**(`openvpn.exe`)을 설치하고 강제 구동하여 장시간 암호화 터널링 링크를 활성화했습니다. 사건 조사 당일 공격자는 VPN 설정 프로필과 구동 흔적 파일들을 일괄 소거하고 접속을 종료한 상태입니다. 하지만 윈도우 시스템은 응용 프로그램이 구동될 때마다 실행 최적화를 목적으로 가동 시간대를 기록 보존하는 프리페치 파일(`.pf`) 내부에 **최종 가동 타임스탬프 이력 목록(Last Execution Timestamps)**을 영구 직렬화해 둡니다. 확보한 OpenVPN 프리페치 파일인 `OPENVPN.EXE-D9E2F1A4.pf`를 파싱 분석하여, **이 비인가 우회 연결 툴이 마지막으로 가동되었던 정확한 최종 실행 시각(플래그)**을 특정하십시오.

## 2. 제공 파일
* `OPENVPN.EXE-D9E2F1A4.pf` (사용자 PC에서 수집한 OpenVPN 실행 대응 윈도우 프리페치 바이너리 파일)

## 3. 문제 목표
윈도우 프리페치(Prefetch) 파일의 세부 데이터 스펙(윈도우 8/10/11 이상에서 최대 8개까지 백업 적재되는 응용 프로그램 최종 실행 시각 배열 구조)을 이해하고, 전문 프리페치 파서 도구(PECmd 등)를 활용하여 정렬된 구동 타임라인을 획득합니다.

## 4. 의도한 풀이 흐름
1. **프리페치 아티팩트 파싱**:
   * 제공된 `OPENVPN.EXE-D9E2F1A4.pf` 파일을 분석하기 위해 Eric Zimmerman의 `PECmd.exe` 도구를 구동합니다:
     `PECmd.exe -f OPENVPN.EXE-D9E2F1A4.pf`
   * 혹은 리눅스 커맨드라인 환경에서 `python` 기반의 프리페치 파서 스크립트를 사용하여 텍스트 데이터 스트림을 획득합니다.
2. **최종 실행 시각 (Last Run Time) 대조**:
   * 도구가 출력하는 메타데이터 명세를 점검합니다:
     * **실행 파일 이름 (Executable Name)**: `OPENVPN.EXE`
     * **실행 횟수 (Run Count)**: `37`
     * **최종 실행 시각 배열 (Last Run Times)**:
       1. `2026-06-28 10:45:15` (최신)
       2. `2026-06-28 09:30:10`
       3. `2026-06-27 18:20:45`
       ...
3. **최종 플래그 조립**:
   * 가장 최근에 가동된 최종 실행 시각 정보인 `2026-06-28 10:45:15` (UTC 기점) 값을 플래그 형식 포맷(YYYY-MM-DD_HH:MM:SS)에 기입하여 플래그를 도출합니다:
     `picoCTF{2026-06-28_10:45:15}`

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{YYYY-MM-DD_HH:MM:SS}`
* **예시**: `picoCTF{2026-06-28_10:45:15}` (모든 시간 연산은 분석가 타임라인 매핑 혼선을 막기 위해 UTC 기준 값으로 수립하여 대입하도록 제한합니다)

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. Windows 10/11 가상 머신에 OpenVPN 클라이언트를 임포트하고 가동합니다.
  2. 여러 차례 OpenVPN 프로세스를 켜고 끄며 실행 횟수가 누적되도록 작동시키되, 마지막으로 작동시키는 시점의 윈도우 OS 시스템 타임스탬프 설정을 `2026-06-28 10:45:15 UTC`에 정확히 맞춘 상태에서 `openvpn.exe`를 구동합니다.
  3. `C:\Windows\Prefetch\` 경로로 진입하여 타임스탬프가 갱신된 `OPENVPN.EXE-XXXXXXXX.pf` 파일을 안전 복제하여 적출합니다.
  4. 적출한 바이너리를 학생 수검용 파일로 배포 포장합니다.
* **출제 포인트**: 
  * 네트워크 우회 수단(VPN, Tor 등)을 이용해 기밀을 유출하고 아티팩트를 소거하는 보안 사고 침해 조사 시, OS 수준에서 강제 로깅하는 최적화 캐시(Windows Prefetch Last Run Times)의 타임라인 타임스탬프 배열 속성을 파싱하여 안티 포렌식 위협을 정면 돌파하는 증적 입증 실무를 학습시킵니다.

## 7. 트러블슈팅 및 힌트
* **Q. 윈도우 버전에 따라 보존되는 실행 시간의 개수가 다른가요?**
  * A. 네, 그렇습니다. 구형 Windows XP/7 환경에서는 실행 빈도가 갱신될 때 오직 최종 실행된 **단 1개**의 타임스탬프만 프리페치 헤더 영역에 덮어씌워 보존되었습니다. 하지만 Windows 8/10/11 이상 계열에서는 시스템 성능 및 감사 정책이 상향되어 실행 기록 배열 구조 슬롯이 늘어남에 따라, 역순으로 정렬된 **최대 8개**의 최근 실행 타임스탬프 기록을 동시 보존하므로 최근 일주일간의 주기적 공격 이력을 입증하는 데 매우 탁월합니다.
* **Q. 프리페치가 비활성화되어 있는 가상 서버 환경에서는 시간 흔적을 어떻게 찾나요?**
  * A. 일반적으로 Windows Server OS 계열(2016, 2019, 2022 등)은 서버 하드웨어 자원 효율화를 위해 프리페치 드라이버 작동 설정(Registry - `EnablePrefetcher`)이 기본값 `0` (비활성)으로 꺼져 있는 경우가 흔합니다. 이럴 때는 레지스트리 기반의 호환성 흔적인 `AppCompatCache` (Shimcache), 또는 `Amcache.hve` 레지스트리의 최종 수정 시간 속성을 대조하여 동일 목적의 가동 흔적 시간을 추적해야 합니다.

## 8. 학습 포인트
* **윈도우 프리페치(Prefetch) 파일 헤더 저장 구조**: 프리페치 바이너리 레이아웃 구조 내에 직렬화 할당되는 8개 다중 실행 타임스탬프 스펙을 이해합니다.
* **비인가 VPN 우회 통신 역추적**: 보안 모니터링을 우회하는 가상 터널 기동에 대한 마지막 구동 순간을 프리페치 시간 타임라인 분석을 통해 수사적으로 증명하는 기법을 마스터합니다.

# 리눅스 배시 환경변수 조작 역추적 (Linux Bash Startup Scripts)

> **난이도**: 초급  
> **소요 시간**: 15~20분  
> **참고 picoCTF 문제**: 사용자 배시 환경 설정 변조 및 영속성 탐지 (.bashrc 하이재킹 분석)

## 1. 배경 시나리오
특정 내부 엔지니어 계정이 원격 SSH로 원격 터미널 로그인 세션을 수립할 때마다, 비정상적인 백그라운드 프로세스가 트리거되어 로컬 기밀 환경변수를 릴레이 수집하는 악성 흔적이 탐지되었습니다. 침해사고 대응팀은 공격자가 시스템 재부팅이나 별도의 크론탭 작동 없이도 사용자가 로그인할 때마다 백도어 스크립트가 로딩되도록 해당 계정의 **배시 초기화 설정 파일(`.bashrc`)**을 은밀히 변조한 정황을 포착했습니다. 적출한 사용자의 `.bashrc` 설정 파일을 정적 분석하여 **공격자가 스크립트 최하단부에 은밀히 삽입해 놓은 해독 페이로드(플래그)**를 획득하십시오.

## 2. 제공 파일
* `.bashrc` (용의자 사용자 홈 디렉터리에서 수집한 변조된 배시 셸 설정 구성 파일)

## 3. 문제 목표
리눅스 사용자가 대화형 셸(Interactive Shell)을 시작할 때 로딩되는 구성 파일(`.bash_profile`, `.profile`, `.bashrc`)의 적용 우선순위와 영속성(Persistence) 유지 메커니즘을 이해하고, 파일 후반부에 삽입된 악성 난독화 구문을 추출하여 해독합니다.

## 4. 의도한 풀이 흐름
1. **정적 파일 정밀 검사**:
   * 제공된 `.bashrc` 파일을 텍스트 뷰어로 엽니다.
   * 공격자들은 일반적으로 기존 표준 배시 설정(약 100~150줄 내외) 뒤에 자신의 악성 구문을 숨기므로, 파일의 맨 하단(Tail) 영역으로 스크롤하여 이동합니다.
2. **의심스러운 추가 행 분석**:
   * 파일의 가장 마지막 라인에서 시스템 기본 구문과 동떨어진 아래의 난독화 코드를 발견합니다:
     ```bash
     # ... [기존 .bashrc 설정들]
     
     # System Update Daemon Helper
     eval $(echo cGljb0NURntiYXNocmNfc3RhcnR1cF9oMWo0Y2tfZGV0M2N0ZWR9 | base64 -d)
     ```
3. **페이로드 디코딩 및 플래그 획득**:
   * `eval` 명령어로 자동 런타임 실행되도록 연계된 Base64 인코딩 페이로드 문자열(`cGljb0NURntiYXNocmNfc3RhcnR1cF9oMWo0Y2tfZGV0M2N0ZWR9`)을 분리해 냅니다.
   * Base64 디코딩 명령어를 수행합니다:
     `echo "cGljb0NURntiYXNocmNfc3RhcnR1cF9oMWo0Y2tfZGV0M2N0ZWR9" | base64 -d`
   * 최종 복구된 플래그를 확인합니다:
     `picoCTF{bashrc_startup_h1j4ck_det3cted}`

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<flag_string>}`
* **예시**: `picoCTF{bashrc_startup_h1j4ck_det3cted}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 기밀 플래그 `picoCTF{bashrc_startup_h1j4ck_det3cted}`를 Base64 인코딩하여 인자 값을 도출합니다.
  2. 우분투 환경의 표준 기본 `.bashrc` 파일을 획득합니다.
  3. 획득한 파일 맨 하단에 공격자 시나리오의 자동 구동 백도어 구문(예: `eval $(echo ... | base64 -d)`)을 수동으로 편집 기입합니다.
  4. 수정된 `.bashrc` 파일을 포장하여 배포용 챌린지 파일로 사용합니다.
* **출제 포인트**: 
  * 리눅스 침해 사고 분석 시 가장 빈번히 오용되는 셸 초기 구동 프로필 하이재킹(Shell Profile Hijacking) 기법을 식별하고, 시스템 관리 쉘 환경 설정 파일에 은밀히 은닉된 비정상적인 명령행 매개변수를 격리 및 해독하는 감사 프로세스를 훈련합니다.

## 7. 트러블슈팅 및 힌트
* **Q. 용의자가 .bash_profile이나 /etc/profile.d/ 하위에 백도어를 심는 것과 어떤 차이가 있나요?**
  * A. `.bash_profile`이나 `/etc/profile`은 사용자가 터미널에 아이디/패스워드를 치고 들어오는 최초 '로그인 셸(Login Shell)' 시점에 단 한 번 로딩됩니다. 반면 `.bashrc`는 로그인 완료 이후 터미널 창을 하나 더 열거나 서브 셸 세션을 구동할 때마다 수시로 기동되는 '비로그인 대화형 셸(Non-login Interactive Shell)' 동작 시 로딩되므로 백도어의 상주 및 트리거 빈도가 훨씬 더 조밀합니다.
* **Q. eval 명령어를 분석 환경에서 실제로 구동해 봐도 안전한가요?**
  * A. 본 챌린지의 페이로드는 단순 플래그 텍스트이나, 실무 침해 사고 현장에서는 `eval`에 실제 원격 포트를 개방하는 리버스 커넥션 셸이나 악성 ELF 바이너리 런처가 연결되어 있을 수 있으므로, 절대로 해당 스크립트를 분석 시스템에서 직접 기동하지 말고 strings 분석 및 base64 문자 디코딩만을 정적으로 선행 적용해야 안전합니다.

## 8. 학습 포인트
* **리눅스 셸 로딩 라이프사이클**: 사용자가 원격/로컬 터미널을 개방해 셸 세션을 점유할 때 커널과 셸 인터프리터가 초기 구성을 위해 호출하는 설정 파일들의 호출 순서와 특징을 명확히 구분합니다.
* **영속성 구성 탐지**: 악성코드 차단 후에도 계정 프로필 변조를 통해 시스템에 재침투하는 백도어 상주 흔적을 정적 구성 감사 파일 비교를 통해 검출해 내는 원리를 습득합니다.

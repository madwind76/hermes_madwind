# 리눅스 배시 환경변수 이진 은닉 해독 v2 (Linux Bash Startup Script Bytecode v2)

> **난이도**: 초급  
> **소요 시간**: 15~20분  
> **참고 picoCTF 문제**: 배시 스타트업 스크립트 8진수 난독화 해독 (Linux Bash Octal Escaping 분석)

## 1. 배경 시나리오
공격자가 리눅스 웹서버에 영속적인 관리자 권한 연결을 유지하고 자동 구동하기 위해 사용자 프로필 설정 파일(`.profile`) 하단에 백도어 트리거 코드를 숨겨두었습니다. 이번에는 보안 검색기들이 16진수 문자열(`\xHH`)을 필터링하는 룰셋을 완벽히 비껴가기 위해, 명령어 전체를 8진수 아스키 이스케이프 포맷(`\OOO`)으로 치환한 뒤 `printf`와 `eval` 함수를 연계하여 기동되도록 난독화를 적용해 두었습니다. 피해 시스템의 스타트업 감사 중 확보한 복구 스크립트 조각 파일 `bash_startup.sh`를 정적 분석하여 **8진수 이스케이프 문자열 속에 마스킹되어 있던 원본 플래그**를 획득하십시오.

## 2. 제공 파일
* `bash_startup.sh` (사용자 프로필 설정 파일 하단에서 회수한 난독화된 셸 스크립트 코드)

## 3. 문제 목표
보안 관제 회피 목적으로 활용도가 점증하는 8진수 이스케이프(`\OOO` 또는 `\0OOO`) 디코딩 메커니즘을 이해하고, 격리 터미널 환경에서 코드를 구동하지 않고 오직 문자열을 수작업 복원하여 플래그를 추출합니다.

## 4. 의도한 풀이 흐름
1. **스크립트 코드 검사**:
   * 제공된 `bash_startup.sh` 파일을 텍스트 뷰어로 엽니다.
   * 파일 내부의 난독화 구문을 식별합니다:
     `eval $(printf '\145\143\150\157\040\042\160\151\143\157\103\124\106\173\142\141\163\150\137\157\143\164\141\154\137\160\141\171\154\157\141\144\137\144\145\143\157\144\145\144\175\042')`
2. **8진수 이스케이프 스트림 격리**:
   * `printf` 함수의 인자로 지정된 8진수 문자열 시퀀스를 추출합니다:
     `\145\143\150\157\040\042\160\151\143\157\103\124\106\173\142\141\163\150\137\157\143\164\141\154\137\160\141\171\154\157\141\144\137\144\145\143\157\144\145\144\175\042`
3. **8진수 아스키 디코딩**:
   * **CLI 디코딩**: 터미널 셸에서 `eval` 명령을 제외하고 `printf` 바이트 스트림만 안전하게 출력시킵니다:
     `printf '\145\143\150\157\040\042\160\151\143\157\103\124\106\173\142\141\163\150\137\157\143\164\141\154\137\160\141\171\154\157\141\144\137\144\145\143\157\144\145\144\175\042'`
   * 출력 결과 복원된 명령어를 확인합니다:
     `echo "picoCTF{bash_octal_payload_decoded}"`
4. **플래그 도출**:
   * 해독 결과 출력된 텍스트로부터 플래그 값을 획득합니다:
     `picoCTF{bash_octal_payload_decoded}`

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<flag_string>}`
* **예시**: `picoCTF{bash_octal_payload_decoded}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 기밀 명령어 구문 `echo "picoCTF{bash_octal_payload_decoded}"` 문자열의 각 문자 아스키 코드를 8진수 표현법(Octal)으로 변환합니다:
     * `e` -> 10진수 `101` -> 8진수 `145` -> `\145`
     * `c` -> 10진수 `99` -> 8진수 `143` -> `\143`
     * `h` -> 10진수 `104` -> 8진수 `150` -> `\150`
     * `o` -> 10진수 `111` -> 8진수 `157` -> `\157`
  2. 얻은 8진수 변환 값을 순차 매핑하여 셸의 `eval $(printf '...')` 구조로 구성합니다.
  3. 스크립트 코드를 `bash_startup.sh` 파일로 최종 저장하여 수검용 아티팩트로 지정합니다.
* **출제 포인트**: 
  * 배시 셸이 16진수 외에도 백슬래시 기반 8진수 형식의 이진 코드 파싱 기능을 표준으로 지원하는 특성을 인지하고, 악성 스크립트 정적 패턴 탐지를 기만하는 이진 우회 스키마(Octal Obfuscation Evasion)를 분석가가 정밀하게 교정 복구하는 역량을 배양합니다.

## 7. 트러블슈팅 및 힌트
* **Q. printf와 echo 함수에서 8진수 이스케이프 포맷은 어떤 차이가 있나요?**
  * A. 리눅스 `printf` 명령어는 백슬래시 뒤에 바로 1~3자리의 8진수 숫자(`\OOO`)를 적어주면 이를 기본 아스키 문자로 치환 출력합니다. 반면, `echo` 명령어는 시스템 셸 환경에 따라 기본적으로 백슬래시 해석이 꺼져 있는 경우가 많아, 동일한 8진수 해독 결과를 보려면 `echo -e '\145\143...'` 와 같이 `-e` (enable interpretation of backslash escapes) 옵션을 수동 선언해 주어야 정상 동작하므로 디코딩 시 유틸리티의 특성을 유의해야 합니다.
* **Q. 파이썬 스크립트에서 이와 같은 8진수 문자열을 디코딩하여 안전하게 추출하려면 어떻게 하나요?**
  * A. 파이썬의 `ast.literal_eval` 함수나 8진수 매칭 정규식을 돌려 처리할 수 있습니다:
     ```python
     # 파이썬 정규식을 통한 8진수 문자열 변환
     octal_str = r"\145\143\150\157"
     decoded = "".join([chr(int(x, 8)) for x in octal_str.split('\\')[1:] if x])
     print(decoded) # 출력 결과: echo
     ```
     위와 같이 구성하면 가상 OS 셸 인터프리터 가동 부하 없이 안전하게 아스키 문자를 환원할 수 있어 편리합니다.

## 8. 학습 포인트
* **배시 셸 환경변수 이스케이프 명세**: 8진수 아스키 문자셋 구성 및 `printf` 연동 처리 원리를 배웁니다.
* **8진수 난독화(Octal Obfuscation) 극복**: 16진수 필터를 우회하기 위해 작성된 8진수 코드 페이로드를 격리 디코딩 연산을 통해 안전하게 역파싱해 획득하는 수사적 분석 능력을 다집니다.

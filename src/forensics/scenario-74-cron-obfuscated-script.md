# 리눅스 크론 영속성 스크립트 이진 난독화 분석 (Linux Cron Startup Script Hex Escaped)

> **난이도**: 초급  
> **소요 시간**: 15~20분  
> **참고 picoCTF 문제**: 크론탭 영속성 스크립트 난독화 해독 (Linux Cron Script Hex Escaping 분석)

## 1. 배경 시나리오
공격자가 리눅스 웹서버에 영속성을 수립하기 위해 크론 작업(Cron Job)을 등록한 정황이 발견되었습니다. 보안 진단 도구가 셸 스크립트 내부의 악성 C2 연결 주소나 의심 문자열을 스캔하는 것을 회피하기 위해, 공격자는 실행 코드를 16진수 아스키 이스케이프 포맷(`\xNN`)으로 치환한 뒤 `printf`와 `eval` 함수로 런타임에 동적 복구해 기동되도록 교묘히 난독화해 두었습니다. 침해 사고 분석팀이 수집한 예약 스크립트 파일인 `cron_job.sh`를 분석하여 **이진 이스케이프 코드로 위장 수록해 둔 원본 플래그**를 획득하십시오.

## 2. 제공 파일
* `cron_job.sh` (크론 스케줄러에 연동되어 런타임 호출 작동 중이던 난독화된 백도어 셸 스크립트 파일)

## 3. 문제 목표
보안 관제 우회를 위해 다용되는 셸 스크립트 난독화 기법인 `eval $(printf '\xNN...')` 유형의 작동 메커니즘을 이해하고, 정적 코드를 디코딩하여 플래그를 안정적으로 추출합니다.

## 4. 의도한 풀이 흐름
1. **스크립트 정적 감사**:
   * 제공된 `cron_job.sh` 파일을 텍스트 에디터로 엽니다.
   * 파일 내에 선언된 비정상적인 동적 복구 구문을 포착합니다:
     `eval $(printf '\x65\x63\x68\x6f\x20\x22\x70\x69\x63\x6f\x43\x54\x46\x7b\x63\x72\x6f\x6e\x5f\x68\x65\x78\x5f\x70\x61\x79\x6c\x6f\x61\x64\x5f\x64\x65\x63\x6f\x64\x65\x64\x7d\x22')`
2. **이진 시퀀스 격리**:
   * `printf` 함수의 인자로 기입되어 런타임 실행되는 헥사 문자열 목록을 복제 추출합니다:
     `\x65\x63\x68\x6f\x20\x22\x70\x69\x63\x6f\x43\x54\x46\x7b\x63\x72\x6f\x6e\x5f\x68\x65\x78\x5f\x70\x61\x79\x6c\x6f\x61\x64\x5f\x64\x65\x63\x6f\x64\x65\x64\x7d\x22`
3. **이스케이프 디코딩**:
   * **CLI 디코딩**: 리눅스 터미널 셸에서 `eval` 명령을 제외하고 `printf` 바이트 스트림만 안전하게 출력시킵니다:
     `printf '\x65\x63\x68\x6f\x20\x22\x70\x69\x63\x6f\x43\x54\x46\x7b\x63\x72\x6f\x6e\x5f\x68\x65\x78\x5f\x70\x61\x79\x6c\x6f\x61\x64\x5f\x64\x65\x63\x6f\x64\x65\x64\x7d\x22'`
   * 출력 결과 해독된 명령어를 대조합니다:
     `echo "picoCTF{cron_hex_payload_decoded}"`
4. **플래그 도출**:
   * 디코딩 결과 도출된 문자열로부터 플래그 값을 획득합니다:
     `picoCTF{cron_hex_payload_decoded}`

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<flag_string>}`
* **예시**: `picoCTF{cron_hex_payload_decoded}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 기밀 명령어 구문 `echo "picoCTF{cron_hex_payload_decoded}"` 문자열의 각 바이트 문자를 아스키 16진수 문자(`\x65\x63...`) 형식으로 변환 빌드합니다.
  2. 셸 스크립트 작성 양식에 맞춰 `eval $(printf '...')` 구조로 해당 헥사 값을 매핑 수록합니다.
  3. 빌드된 구문을 `cron_job.sh` 파일로 최종 저장하여 수검용 문제 파일로 지정합니다.
* **출제 포인트**: 
  * 리눅스 백그라운드 스케줄 기동 영속성(Cron Persistence Auditing) 분석 과정에서, 파일 평문 시그니처 정적 매칭을 무력화하기 위한 바이너리 변환 난독화 우회 기술을 판별하고, 안전한 격리 환경 하에서 기계적으로 복원 및 해독해 내는 분석 기초 능력을 기릅니다.

## 7. 트러블슈팅 및 힌트
* **Q. 난독화된 셸 코드를 분석 시스템에서 직접 sh cron_job.sh 명령어로 구동해도 무방한가요?**
  * A. 절대 안 됩니다. 본 문제의 페이로드는 단순 플래그를 이중 출력(`echo`)하는 용도이지만, 실제 해킹 공격 코드는 시스템 파일을 소거하거나 외부 원격 연결 세션을 수립하는 파괴적인 명령이 `eval`에 결합되어 있을 수 있습니다. 따라서 절대로 스크립트를 통째로 구동하지 말고, `eval` 문장을 제거한 내부 `printf` 문자열 해독만을 정적으로 적용해 분석해야 안전이 보장됩니다.
* **Q. 이와 같은 백슬래시 이스케이프 포맷을 파이썬 스크립트 상에서 정적으로 치환 디코딩하려면 어떻게 하나요?**
  * A. 파이썬의 `codecs` 모듈을 연동해 해결할 수 있습니다:
     ```python
     import codecs
     raw_escaped = r"\x65\x63\x68\x6f"
     # 백슬래시 제어 문자를 디코드 처리
     decoded_str = codecs.escape_decode(bytes(raw_escaped, "utf-8"))[0].decode("utf-8")
     print(decoded_str)
     ```
     위와 같이 구성하면 가상 OS 셸 인터프리터의 런타임 부하 없이도 문자열만 안전하게 추출해 볼 수 있습니다.

## 8. 학습 포인트
* **크론탭 영속성 스크립트 구조**: 시스템에 잔재하는 자동 예약 스케줄 기반 악성 코드의 주입 특징과 배치 정책을 학습합니다.
* **셸 명령어 난독화(Obfuscation) 극복**: `printf` 와 `eval` API 조합을 악용하여 정적 보안 탐지를 회피하려는 악성 셸 코드를 무력화하고 평문 복구하는 이론을 완벽히 구축합니다.

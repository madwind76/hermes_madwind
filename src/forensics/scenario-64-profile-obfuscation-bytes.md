# 리눅스 배시 환경변수 이진 은닉 해독 (Linux Bash Startup Script Bytecode)

> **난이도**: 초급  
> **소요 시간**: 15~20분  
> **참고 picoCTF 문제**: 로그인 셸 구동 변조 및 우회 분석 (Linux Login Shell Profile 난독화)

## 1. 배경 시나리오
공격자가 사내 핵심 엔지니어의 계정에 임시 침투한 뒤, 단순 검색 필터링(예: `grep` 명령을 이용해 소스코드 내 "picoCTF" 키워드를 색인하는 위협 탐지 패턴)을 우회하기 위해 **로그인 초기 프로필 설정 파일(`.profile`)**에 난독화된 문자열을 삽입해 두었습니다. 공격자는 사용자가 SSH 로그인 세션을 열 때마다 기밀 환경 변수를 은밀하게 복원하여 로컬 변수에 보존하는 스크립트를 작성해 영속성을 유지하려 했습니다. 획득한 용의자 계정의 `.profile` 파일을 검토하여 **16진수 이진(Hex) 제어 마커로 은닉된 플래그**를 해독하십시오.

## 2. 제공 파일
* `.profile` (용의자 사용자 홈 디렉터리에서 획득한 로그인 프로필 환경 설정 파일)

## 3. 문제 목표
리눅스 사용자가 대화형 로그인 셸(Interactive Login Shell)을 획득할 때 기동하는 초기 스크립트(`.profile`) 호출 규칙과 특징을 이해하고, 바이너리 우회용으로 자주 오용되는 16진수 이진 이스케이프 시퀀스(`\xNN` 포맷)를 수동 또는 자동 디코딩하여 플래그를 정상 복원합니다.

## 4. 의도한 풀이 흐름
1. **정적 파일 감사**:
   * 제공된 `.profile` 텍스트 파일을 에디터로 엽니다.
   * 파일 하단부에 이르러, 시스템 기본 셸 지시자 외에 다음과 같이 변수가 동적 할당되는 의심스러운 행을 식별합니다:
     `export HIDDEN_TOKEN=$(printf '\x70\x69\x63\x6f\x43\x54\x46\x7b\x70\x72\x6f\x66\x69\x6c\x65\x5f\x68\x65\x78\x5f\x65\x73\x63\x61\x70\x65\x5f\x64\x65\x63\x6f\x64\x65\x64\x7d')`
2. **난독화 구조 분석**:
   * 공격자는 보안 장비가 텍스트 평문을 탐색하는 것을 피하기 위해 문자열을 16진수 문자(`\x70` = 'p', `\x69` = 'i', `\x63` = 'c' 등)로 조각내어 `printf` 명령어로 실시간 결합하는 형식을 취했습니다.
3. **이스케이프 시퀀스 해독**:
   * **터미널 분석**: 파이프라인 연산을 통해 `printf` 내부의 이스케이프 포맷 문자열을 복제하여 셸에서 직접 안전하게 디코딩을 수행합니다:
     `printf '\x70\x69\x63\x6f\x43\x54\x46\x7b\x70\x72\x6f\x66\x69\x6c\x65\x5f\x68\x65\x78\x5f\x65\x73\x63\x61\x70\x65\x5f\x64\x65\x63\x6f\x64\x65\x64\x7d'`
   * **수동 변환**: CyberChef의 `From Hex` 레시피를 이용하거나 파이썬 셸에서 바이트 해독을 가동합니다:
     `bytes.fromhex('7069636f4354467b70726f66696c655f6865785f6573636170655f6465636f6465647d').decode()`
4. **플래그 도출**:
   * 디코딩된 최종 아스키 문자열인 플래그를 획득합니다:
     `picoCTF{profile_hex_escape_decoded}`

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<flag_string>}`
* **예시**: `picoCTF{profile_hex_escape_decoded}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 기밀 플래그 `picoCTF{profile_hex_escape_decoded}`를 각 바이트마다 16진수 아스키 문자열로 치환하여 `\x70\x69...` 시퀀스를 빌드합니다.
  2. 우분투 환경의 오리지널 기본 `.profile` 템플릿 파일 하단에 `export HIDDEN_TOKEN=$(printf '...')` 구문을 임베딩 편집 기입합니다.
  3. 수정 완료된 파일을 `.profile` 파일 이름으로 챌린지 배포용 파일로 설정합니다.
* **출제 포인트**: 
  * 로그인 시점에 셸 인터프리터 수준에서 자동 실행되는 로그인 셸 프로필 아티팩트(Login Shell Profiles)의 존재를 교육하고, 평문 검색 패턴 감지를 우회하기 위해 다용되는 16진수 포맷 난독화 구문을 가볍게 우회 격격하는 포렌식 기본 분석을 훈련합니다.

## 7. 트러블슈팅 및 힌트
* **Q. .bashrc와 .profile 중 어떤 것이 먼저 실행되나요?**
  * A. 사용자가 SSH 로그인 등을 통해 '로그인 셸(Login Shell)'을 획득할 때 리눅스 커널과 배시 엔진은 우선 `/etc/profile`을 읽은 뒤, 사용자 프로필인 `~/.bash_profile`, `~/.bash_login`, `~/.profile` 순서로 탐색하여 가장 먼저 발견되는 단 한 개의 파일만 호출 및 기동시킵니다. 이후 일반 터미널 조작 시 `~/.bashrc`를 추가 연동 로딩하므로, 침해 조사 시에는 이 프로필 파일 목록 전체의 최종 수정 일시(MACB)와 하단 편집 여부를 통합 대조하여 분석해야 빠짐이 없습니다.
* **Q. printf 대신 echo를 써도 동일한 동작이 수행되나요?**
  * A. 네, `echo -e` 옵션을 인가하면 동일한 백슬래시 이스케이프 해독 동작을 지원하지만, 셸 인터프리터 버전에 따라 `-e` 플래그 기본값 활성화 규칙이 다를 수 있어 스크립트 우회 제작자들은 호환성이 높은 `printf` 내장 함수를 애용합니다.

## 8. 학습 포인트
* **리눅스 로그인 프로필 환경설정 라이프사이클**: 사용자가 원격 시스템 접속을 승인받을 때 백그라운드 호출되는 초기화 파일의 우선순위 사양을 파악합니다.
* **이진 문자 이스케이프 디코딩**: 바이트 데이터를 은닉하기 위해 16진수 헥스 구조로 우회한 난독화 스크립트를 역파싱해 실시간 복구하는 기술적 논리를 함양합니다.

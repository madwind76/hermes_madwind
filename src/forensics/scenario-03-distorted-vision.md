# 찌그러진 뷰포트 (Distorted Vision)

> **난이도**: 초중급  
> **소요 시간**: 25~30분  
> **참고 picoCTF 문제**: `tunn3l v1s10n` (picoCTF 2021)

## 1. 배경 시나리오
사고 대응 부서는 기밀 문서 유출 사건의 피해 PC에서 외형상 깨진 것처럼 보이는 비트맵 이미지 파일 `corrupted_image.bmp`를 확보했습니다. 기본 사진 뷰어에서 파일을 열려고 시도하면 "손상된 파일" 에러가 발생하거나 매우 얇은 가로선 한 줄만 나타나 내용을 읽을 수 없습니다. 용의자는 이미지 헤더 내부의 차원(Dimension) 메타데이터를 직접 변조해 단서를 은폐하려고 시도한 것으로 파악되었습니다. 

## 2. 제공 파일
* `corrupted_image.bmp` (손상된 형태의 BMP 비트맵 파일)

## 3. 문제 목표
BMP 파일 포맷의 저수준 헤더 구조를 분석하고, 헥스 에디터(Hex Editor)를 사용하여 비정상적으로 수정된 이미지 가로/세로 영역(DIB 헤더의 크기 정보)을 찾아내어 정상 값으로 재구성함으로써 숨겨진 플래그 영역을 온전히 시각화합니다.

## 4. 의도한 풀이 흐름
1. **바이너리 구조 확인**: `xxd`, `010 Editor`, `HxD` 등 편한 헥스 에디터로 `corrupted_image.bmp`를 엽니다.
2. **BMP 헤더 포맷 수동 분석**:
   * BMP 파일 시그니처 `BM` (0x42 0x4D)을 확인합니다.
   * 파일 전체 크기(바이트)와 DIB 헤더 시작 오프셋 위치(일반적으로 0x36 등)를 검사합니다.
3. **차원 값(Dimensions) 불일치 탐지**:
   * BMP의 DIB 헤더인 `BITMAPINFOHEADER` 구조에서 가로(Width, 오프셋 18)와 세로(Height, 오프셋 22) 값을 파싱합니다.
   * 파일의 전체 바이트 크기에 비해 세로(Height) 바이트 값(예: `01 00 00 00` -> 1픽셀)이 터무니없이 작게 조작되어 있음을 인지합니다.
4. **바이너리 값 수정**:
   * 세로(Height)에 해당하는 위치의 4바이트 값을 더 큰 값(예: 300픽셀 혹은 500픽셀 상당인 `F0 01 00 00` 등)으로 변조하고 저장합니다.
5. **이미지 복원 확인**: 수정된 이미지 파일을 일반 뷰어로 열었을 때, 잘려서 보이지 않던 이미지의 본문 및 최하단 영역에 적힌 플래그 문자열을 발견합니다.

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<flag_string>}`
* **예시**: `picoCTF{bmp_header_height_restored}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 포토샵이나 그림판을 사용하여 흰 배경에 플래그를 포함하는 `original.bmp` 이미지를 생성합니다. (예: 가로 800px, 세로 500px, 24비트 비트맵)
  2. 헥스 에디터를 이용하여 생성된 BMP 파일의 오프셋 22(세로 높이) 위치의 값을 `01 00 00 00`(1px) 또는 `0A 00 00 00`(10px) 등으로 덮어써서 실제 이미지가 가늘게만 출력되도록 왜곡합니다.
  3. 오프셋 14 위치의 DIB 헤더 크기(`BITMAPINFOHEADER`인 경우 40바이트)나 오프셋 28 위치의 색상 비트(24비트) 등 기타 값이 정상 범주 내인지 확인하여, 단순히 비트맵 리더 에러가 나지 않고 렌더링만 잘리도록 오프셋 조정을 설계합니다.
* **출제 포인트**: 
  * 파일 포맷의 명세를 이해하고, 헥스 오프셋을 직접 변경하여 손상되거나 은폐된 바이너리를 복구하는 실무 역량을 배양합니다.

## 7. 트러블슈팅 및 힌트
* **Q. 세로 크기를 아무 값이나 크게 넣어도 플래그가 보이나요?**
  * A. 너무 작은 값으로 바꾸면 여전히 이미지가 잘려 보이고, 실제 데이터 범위를 넘어서는 지나치게 큰 값으로 수정하면 이미지 뷰어에 따라 하단에 노이즈가 출력되거나 파일 로드 에러가 발생할 수 있습니다. 원래 픽셀 크기에 가장 가까운 적절한 크기(예: 300~500px)를 계산하거나 시행착오로 범위를 맞추어야 합니다.
* **Q. BMP 헤더 명세는 어디서 보나요?**
  * A. 비트맵 파일 구조에서 처음 14바이트는 `BITMAPFILEHEADER`, 그 다음은 `BITMAPINFOHEADER` (기본 40바이트)가 옵니다. 세로 크기는 파일 시작 지점 기준 오프셋 22(0x16)부터 4바이트 크기로 리틀 엔디언 구조로 저장되어 있습니다.

## 8. 학습 포인트
* **바이너리 헤더 조작**: 공격자가 정보 은닉을 위해 헤더 차원을 축소하여 데이터 카빙을 방해하는 원리와 방어적 분석 기법을 습득합니다.
* **DIB(Device-Independent Bitmap) 명세**: 헤더 필드의 오프셋 매핑 구조를 파악하고, 전체 파일 크기와 픽셀당 비트 수(bpp)의 상관관계를 통해 적절한 이미지 해상도를 역산하는 역량을 학습합니다.

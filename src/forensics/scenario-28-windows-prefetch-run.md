# 윈도우 프리페치 분석 (Windows Prefetch Application Execution)

> **난이도**: 초급  
> **소요 시간**: 15~20분  
> **참고 picoCTF 문제**: 프로그램 실행 흔적 추적 (Windows Prefetch .pf 파일 분석)

## 1. 배경 시나리오
보안 위협 탐지 시스템은 기밀 데이터 유출 의심 시간에 내부 임직원 PC에서 미승인 데이터 유출 툴인 `exfil.exe`가 다수 작동한 정황을 잡아내고 시스템 권한을 정지시켰습니다. 용의자는 해당 실행 파일(`exfil.exe`)을 디스크에서 휴지통도 거치지 않고 완전히 영구 삭제(Shift+Del)하여 관련 흔적을 지웠다고 진술했습니다. 하지만 윈도우 운영체제의 프로그램 실행 고속화 기능인 **프리페치(Prefetch)** 디렉터리(`C:\Windows\Prefetch\`) 하위에서 해당 실행 파일의 캐싱 아티팩트인 `EXFIL.EXE-3A2B1C2D.pf` 파일이 원본 그대로 복구되었습니다. 이 프리페치 파일을 파싱하여 **참조된 로딩 파일 목록 중 특이 파일명에 숨어 있는 플래그**를 획득해야 합니다.

## 2. 제공 파일
* `EXFIL.EXE-3A2B1C2D.pf` (Windows 10/11 규격의 프리페치 바이너리 파일)

## 3. 문제 목표
윈도우 프리페치 파일의 생성 목적과 데이터 저장 명세(실행 횟수, 최종 실행 시간, 로딩된 볼륨 및 참조 파일 목록)를 이해하고, 전문 프리페치 파서 도구(PECmd 등)를 활용하여 내부 파일 로딩 리스트를 추출해 숨겨진 플래그를 찾아냅니다.

## 4. 의도한 풀이 흐름
1. **아티팩트 특성 진단**:
   * 제공된 파일의 확장자가 `.pf`이며 Windows 10/11 시스템인 경우 기본적으로 LZX 방식의 독자적인 **MAM 압축(Header: `MAM\x04`)** 구조로 캡슐화되어 있음을 인지합니다.
2. **프리페치 파일 파싱**:
   * **CLI 도구 활용 (PECmd)**: Eric Zimmerman의 프리페치 분석 도구인 `PECmd.exe`를 사용하여 분석합니다.
     `PECmd.exe -f EXFIL.EXE-3A2B1C2D.pf`
   * **Linux/Python 환경**: `python-prefetch` 등 오픈소스 파서를 활용해 원시 데이터를 텍스트 형태로 디코딩하여 덤프합니다.
3. **참조 리소스(Files Referenced) 분석**:
   * 파싱 결과로 도출되는 프로그램 메타데이터를 파악합니다:
     * `Last run time`: 최종 실행 시간
     * `Run count`: 실행 횟수
     * `Files referenced`: 프로그램 구동 시 메모리에 적재하며 참조했던 DLL 및 데이터 파일 목록 (일반적으로 50~150개 내외)
   * 참조 파일 경로 목록을 하나하나 살피는 도중, 사용자 문서 경로 하위에 기록된 비정상적인 형태의 파일명을 발견합니다:
     `\VOLUME{01d0a...}\USERS\ADMIN\DOCUMENTS\FLAG_picoCTF{pr3f3tch_r3veals_execut1on_h1st0ry}.TXT`
4. **플래그 식별**:
   * 발견된 파일명에서 플래그 문자열 부분을 식별하여 제출합니다:
     `picoCTF{pr3f3tch_r3veals_execut1on_h1st0ry}`

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<flag_string>}`
* **예시**: `picoCTF{pr3f3tch_r3veals_execut1on_h1st0ry}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 임의의 C언어 코드 또는 스크립트를 빌드하여 `exfil.exe`를 작성합니다. 이 프로그램은 구동 시 특정 경로의 파일을 의도적으로 오픈(`fopen`)하는 행위를 수행합니다:
     `fopen("C:\\Users\\admin\\Documents\\FLAG_picoCTF{pr3f3tch_r3veals_execut1on_h1st0ry}.txt", "r")`
  2. Windows 10/11 가상 환경에서 위 프로그램을 실행합니다. 프로그램 실행 즉시 윈도우 OS 커널은 10초간의 디스크 I/O 동작을 감시하여 참조된 파일 목록을 `EXFIL.EXE-<해시값>.pf` 파일로 자동 기록 및 압축합니다.
  3. `C:\Windows\Prefetch\` 경로로 이동하여 생성된 `.pf` 파일을 획득합니다.
  4. 획득한 프리페치 파일을 단독 파일 형태로 정제하여 배포합니다.
* **출제 포인트**: 
  * 프로그램 실행 증적(Execution Artifacts)의 중요성과 프리페치 아티팩트 분석 도구 활용 능력을 평가하고, 실행 파일이 삭제되더라도 윈도우 커널 수준에서 참조 흔적이 RAM 기호를 포함해 고스란히 저장되는 메커니즘을 증명합니다.

## 7. 트러블슈팅 및 힌트
* **Q. 텍스트 에디터로 열었을 때 파일 앞부분이 MAM... 으로 깨져 나옵니다.**
  * A. Windows 8.1/10 이후 버전부터는 프리페치 파일의 디스크 공간 절약을 위해 독자적인 MAM 압축 포맷을 적용합니다. 텍스트 에디터로는 내부의 파일 목록 문자열을 가독할 수 없으므로, 반드시 `PECmd`와 같은 전용 파서 도구로 압축을 임시 해제하며 파싱해야 합니다.
* **Q. 실행 파일 해시값(예: 3A2B1C2D)은 어떻게 생성되나요?**
  * A. 프리페치 파일명 뒤에 붙는 8자리 헥스 해시는 실행 파일의 절대 경로와 커맨드라인 매개변수 정보를 바탕으로 계산되는 특유의 윈도우 알고리즘 해시값입니다. 실행 파일이 위치한 경로가 다르면 이 해시값도 변경됩니다.

## 8. 학습 포인트
* **윈도우 프리페치(Prefetch) 명세**: 프로그램 구동 속도 향상을 위한 메모리 모니터링 캐싱 아티팩트의 구조를 파악합니다.
* **프로그램 실행 증적 조사**: 시스템 내 특정 툴의 실행 유무, 횟수, 시점, 그리고 구동 과정에서 로딩해 간 주변 데이터 파일의 전체 경로를 역추적해 내는 침해 사고 정밀 추적(DFIR) 기법을 학습합니다.

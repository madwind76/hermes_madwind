# 이중 언어 파일 (Polyglot Artifact)

> **난이도**: 초급  
> **소요 시간**: 15~20분  
> **참고 picoCTF 문제**: `Secret of the Polyglot` (picoCTF 2024)

## 1. 배경 시나리오
사내 연구원의 PC가 침해되어 보안 부서가 대응에 착수했습니다. 공격자가 데이터를 탈취하면서 공격 도구를 숨겨 둔 파일 `flag_carrier`를 발견했습니다. 이 파일은 더블 클릭하여 실행하면 평범한 로고 이미지로 보이지만, 파일의 메타데이터를 정밀 점검한 보고서에 따르면 PDF 포맷의 문서 파일 매직 바이트 시그니처도 하단에 동시 검출되고 있습니다. 이 파일의 이중 속성(Polyglot)을 탐색해 두 포맷에 나뉘어 숨겨진 플래그를 결합해야 합니다.

## 2. 제공 파일
* `flag_carrier` (확장자가 제공되지 않은 다중 구조 파일)

## 3. 문제 목표
하나의 파일에 두 가지 이상의 정상 파일 시그니처가 병합되어 있는 폴리글로트(Polyglot) 특성을 확인하고, 이미지 뷰어와 PDF 리더 두 개의 경로로 파일을 해석해 각각에 기록된 플래그 조각을 찾아 조립합니다.

## 4. 의도한 풀이 흐름
1. **파일 구조 분석**:
   * 리눅스 터미널에서 `file flag_carrier` 명령을 사용해 파일의 형식을 확인합니다.
   * 첫 번째 확인 결과 PNG 또는 PDF 형식으로 나타날 것입니다.
2. **첫 번째 조각 획득 (이미지 분석)**:
   * 파일 확장자를 `.png`로 추가하여 `flag_carrier.png`로 저장한 뒤 이미지 뷰어로 엽니다.
   * 이미지 영역 안에 텍스트 형식으로 플래그의 전반부 조각이 그려져 있는 것을 식별합니다.
     (예: `picoCTF{p0lygl0t_`)
3. **두 번째 조각 획득 (PDF 분석)**:
   * 헥스 에디터 등으로 검사하여 파일 하단에 PDF 문서 시그니처(`%PDF-`) 및 트레일러가 공존하고 있음을 확인합니다.
   * 파일 확장자를 `.pdf`로 변경하고 PDF 뷰어 프로그램(또는 웹 브라우저)으로 엽니다.
   * PDF 문서 내용 안에 텍스트로 적혀 있는 플래그 후반부 조각을 발견합니다.
     (예: `f1l3s_4r3_h3r3}`)
4. **플래그 조립**: 두 플래그 조각을 꼬리에 맞게 결합하여 정답을 완성합니다.
   (최종 플래그: `picoCTF{p0lygl0t_f1l3s_4r3_h3r3}`)

## 5. 정답(플래그) 규칙 및 예시
* **플래그 형식**: `picoCTF{<flag_string>}`
* **예시**: `picoCTF{p0lygl0t_f1l3s_4r3_h3r3}`

## 6. 출제자 노트 & 파일 제작 가이드
* **아티팩트 제작 방법**:
  1. 첫 번째 조각(`picoCTF{p0lygl0t_`)이 그려진 작은 PNG 이미지 `part1.png`를 만듭니다.
  2. 두 번째 조각(`f1l3s_4r3_h3r3}`)이 포함된 1페이지짜리 PDF 문서 `part2.pdf`를 작성합니다.
  3. 두 개의 파일을 단순 병합 명령어로 이어 붙입니다.
     `cat part1.png part2.pdf > flag_carrier`
  4. 파일의 앞부분에는 PNG의 시그니처(`89 50 4E 47`)가 위치하므로 일반 이미지 뷰어는 PNG 파일로 읽고 해석을 정상 종료합니다.
  5. 반면 PDF 파서들은 문서 시작부 이전에 존재하는 가비지 데이터(여기서는 PNG 영역)를 무시하고 바이너리 검색을 통해 `%PDF-` 문자열을 찾아 문서를 구성하므로, PDF 리더에서는 정상적으로 PDF로 열립니다.
* **출제 포인트**: 
  * 파일 뷰어들이 파일 포맷을 파싱할 때 시그니처를 참조하는 관용적인 방식(오프셋 기준, 바이너리 탐색 등)의 허점을 이해하고 이중 시그니처를 활용한 파일 카빙 기법을 실습합니다.

## 7. 트러블슈팅 및 힌트
* **Q. PDF 확장자로 변경하고 열었는데 파일이 깨졌다는 에러가 발생합니다.**
  * A. 사용하시는 PDF 리더에 따라 가비지 데이터(PNG 바이트) 크기가 너무 크면 문서를 손상된 파일로 취약성 판단하여 열지 못할 수 있습니다. 크롬(Chrome) 브라우저에 파일을 던져 렌더링을 시도해 보거나, `strings` 또는 헥스 에디터를 통해 `%PDF`로 시작하는 영역 이후만 별도의 파일로 카빙해 내어 열어보십시오.
* **Q. png 뷰어에서 후반부 플래그는 전혀 보이지 않습니다.**
  * A. 정상적인 PNG 포맷 뷰어는 `IEND` 청크 시그니처를 만나면 이미지 렌더링을 종결하므로, IEND 뒷부분에 추가된 PDF 데이터는 무시됩니다. 따라서 각 포맷 전용의 뷰어로 둘 다 열어야 답이 나옵니다.

## 8. 학습 포인트
* **폴리글로트(Polyglot) 파일**: 하나의 파일이 다중 성격을 가져 정밀 보안 필터링을 우회(예: 이미지만 업로드 가능한 게시판에 PDF/웹셸 업로드)하는 데 악용되는 개념을 학습합니다.
* **바이너리 카빙 역량**: 여러 헤더 시그니처가 존재할 때 수동으로 파일 포맷 단위의 분기를 나누어 카빙해 내는 기초 분석 기법을 습득합니다.

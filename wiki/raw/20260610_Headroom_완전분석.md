## 1. Headroom이란?

**Headroom은 LLM 애플리케이션을 위한 컨텍스트 최적화 레이어다.**

AI 에이전트가 읽는 모든 것 — 툴 출력, 로그, RAG 청크, 파일, 대화 기록 — 을 LLM에 보내기 전에 압축한다. **같은 답변, 토큰은 60-95% 적게.**

```
      Your agent / app
   (Claude Code, Cursor, Codex, LangChain, …)
        │   prompts · tool outputs · logs · RAG results · files
        ▼
    ┌────────────────────────────────────────────────────┐
    │  Headroom   (로컬 실행 — 데이터는 내 머신에)         │
    │  ────────────────────────────────────────────────  │
    │  CacheAligner  →  ContentRouter  →  CCR            │
    │                    ├─ SmartCrusher   (JSON)        │
    │                    ├─ CodeCompressor (AST)         │
    │                    └─ Kompress-base  (text, HF)    │
    │                                                    │
    │  Cross-agent memory  ·  headroom learn  ·  MCP     │
    └────────────────────────────────────────────────────┘
        │   compressed prompt  +  retrieval tool
        ▼
      LLM provider (Anthropic · OpenAI · Bedrock · …)
```

## 2. 핵심 개념

### 2.1 6가지 압축 알고리즘

| 알고리즘 | 대상 | 방식 | 압축률 |
|---------|------|------|--------|
| **SmartCrusher** | JSON 배열 (툴 출력) | 통계적 분석: 에러·이상값·경계만 보존 | 70-90% |
| **CodeCompressor** | 소스코드 | AST 기반: 시그니처 보존, body 압축 | 40-70% |
| **Kompress-base** | 일반 텍스트 | HuggingFace 모델 (ModernBERT token classification) | 30-50% |
| **Image Compression** | 이미지 | ML 라우터로 최적 resize/quality 선택 | 40-90% |
| **CacheAligner** | 프롬프트 접두사 | Anthropic/OpenAI KV 캐시 hit율 최적화 | 간접적 |
| **IntelligentContext** | 전체 컨텍스트 | 중요도 기반 점수화 + learned importance | 상황별 |

### 2.2 CCR (Reversible Compression)

- 압축해도 **원본은 절대 삭제되지 않음**
- LLM이 필요하면 `headroom_retrieve` 툴로 원본 조회 가능
- 투명성 + 효율성 동시 확보

### 2.3 Cross-Agent Memory

- 여러 에이전트(Claude, Codex, Gemini)가 **공유 메모리** 사용
- 자동 중복 제거, 에이전트 출처 추적
- `SharedContext().put / .get` 으로 멀티 에이전트 워크플로 지원

### 2.4 headroom learn

- 실패한 세션을 분석해 `CLAUDE.md` / `AGENTS.md`에 교정사항 자동 작성
- 반복 실패 방지, 학습 효과 극대화

## 3. 4가지 사용 모드

### 모드 A: Agent Wrap (1커맨드)

```bash
pip install "headroom-ai[all]"

# 가장 쉬운 방법 — 코딩 에이전트를 자동 래핑
headroom wrap claude           # Claude Code
headroom wrap codex            # Codex CLI
headroom wrap cursor           # Cursor
headroom wrap aider            # Aider
headroom wrap copilot          # GitHub Copilot CLI
```

**원리**: 환경변수(API URL 등)를 Headroom 프록시로 리다이렉트 → 모든 트래픽이 자동 압축

### 모드 B: Proxy (코드 변경 0)

```bash
# 프록시 서버 실행
headroom proxy --port 8787

# 모든 OpenAI 호환 클라이언트가 이 프록시를 통해 통신
# ANTHROPIC_BASE_URL=http://localhost:8787
# OPENAI_BASE_URL=http://localhost:8787
```

**원리**: LLM API 호출을 프록시가 가로채서 요청 본문 압축 후 원래 API로 전달, 응답 반환

### 모드 C: Library (인라인)

```python
from headroom import compress

messages = [
    {"role": "user", "content": "이 시스템 로그에서 에러를 찾아줘"},
    {"role": "tool",  "content": json.dumps(100개의_로그_항목)},
]

result = compress(messages, model="gpt-4o")
print(f"{result.savings_pct:.1f}% 토큰 절약!")
# 압축된 메시지를 LLM에 그대로 전송
response = client.chat.completions.create(
    model="gpt-4o",
    messages=result.messages,   # ← 압축된 메시지
)
```

```typescript
import { compress } from "headroom-ai";

const result = await compress(messages, { model: "gpt-4o" });
```

### 모드 D: Framework Integration

| 프레임워크 | 통합 방법 |
|-----------|----------|
| **Anthropic/OpenAI SDK** | `withHeadroom(new Anthropic())` |
| **Vercel AI SDK** | `wrapLanguageModel({ model, middleware: headroomMiddleware() })` |
| **LiteLLM** | `litellm.callbacks = [HeadroomCallback()]` |
| **LangChain** | `HeadroomChatModel(your_llm)` |
| **Agno** | `HeadroomAgnoModel(your_model)` |
| **ASGI 앱** | `app.add_middleware(CompressionMiddleware)` |
| **MCP 클라이언트** | `headroom mcp install` |

## 4. 설치 방법

### 기본 설치

```bash
# Python (전체 설치)
pip install "headroom-ai[all]"

# TypeScript / Node
npm install headroom-ai

# Docker
docker pull ghcr.io/chopratejas/headroom:latest
```

### 선택적 extras

| extra | 포함 기능 |
|-------|----------|
| `[all]` | 전부 |
| `[proxy]` | 프록시 서버 |
| `[mcp]` | MCP 서버 도구 |
| `[ml]` | Kompress-base (HuggingFace 모델) |
| `[code]` | CodeCompressor (AST 압축) |
| `[memory]` | Cross-agent memory (Qdrant/Neo4j) |
| `[image]` | 이미지 압축 |
| `[langchain]` | LangChain 통합 |
| `[agno]` | Agno 통합 |
| `[evals]` | 벤치마크 실행 |

```bash
# 필요한 것만 설치 (예: 프록시 + 기본 압축)
pip install "headroom-ai[proxy,ml,code]"
```

### pipx 사용 시

```bash
pipx install --python python3.13 "headroom-ai[all]"
```

**요구사항**: Python 3.10+

## 7. 실제 적용 예시

### 시나리오 1: Claude Code에 Headroom 입히기 (가장 쉬움)

```bash
# 1. 설치
pip install "headroom-ai[all]"

# 2. Claude Code 래핑
headroom wrap claude

# 3. 평소처럼 Claude Code 사용
# Claude → Headroom Proxy → Anthropic API
# 토큰 자동 절약, 별도 설정 불필요
```

### 시나리오 2: LangChain 앱에 Headroom 추가

```python
from headroom.integrations.langchain import HeadroomChatModel
from langchain_openai import ChatOpenAI

original_llm = ChatOpenAI(model="gpt-4o")
compressed_llm = HeadroomChatModel(original_llm)

# 압축된 LLM을 평소처럼 사용
result = compressed_llm.invoke("로그 1000줄에서 에러 찾아줘")
# → Headroom이 자동으로 로그 압축 후 LLM 전송
```

### 시나리오 3: 프록시 모드 (코드 변경 0)

```bash
# 터미널 1: Headroom 프록시 실행
headroom proxy --port 8787

# 터미널 2: 환경변수 설정 후 기존 앱 실행
export OPENAI_BASE_URL=http://localhost:8787
export ANTHROPIC_BASE_URL=http://localhost:8787
python my_app.py   # 코드 변경 전혀 없음
```

### 시나리오 4: MCP 서버로 사용

```bash
# MCP 설정에 Headroom 도구 추가
headroom mcp install

# MCP 클라이언트에서 사용 가능한 도구:
# - headroom_compress: 메시지 압축
# - headroom_retrieve: 원본 복원
# - headroom_stats: 절약 통계
```

## 8. 보안 고려사항

| 측면 | Headroom의 처리 |
|------|----------------|
| **데이터 프라이버시** | **로컬 실행**. 데이터가 내 머신을 벗어나지 않음 |
| **원본 보존** | CCR — 원본 데이터는 로컬에 보관, LLM이 필요시 retrieval |
| **암호화** | 프록시 모드에서 전송中的数据는 TLS 유지 |
| **API 키** | Headroom이 API 키를 보거나 저장하지 않음 — 원래 LLM API로 전달만 |

## 9. Headroom vs 대안

| 도구 | 범위 | 배포 | 로컬 | 복원 가능 |
|------|------|------|:---:|:---------:|
| **Headroom** | 모든 컨텍스트 | Proxy · Library · MCP | ✅ | ✅ (CCR) |
| RTK | CLI 명령 출력 | CLI wrapper | ✅ | ❌ |
| lean-ctx | CLI 명령, MCP, 에디터 규칙 | CLI · MCP | ✅ | ❌ |
| Compresr / Token Co. | 텍스트 | Hosted API | ❌ | ❌ |
| OpenAI Compaction | 대화 기록 | Provider-native | ❌ | ❌ |

> **Headroom의 차별점**: 모든 컨텐츠 타입, 모든 주요 프레임워크, 로컬 실행, 복원 가능

## 10. 언제 쓰고 언제 스킵할까

### 좋은 상황 👍
- AI 코딩 에이전트를 **매일 사용**하고 토큰 비용 아끼고 싶을 때
- 여러 에이전트(Claude + Codex)를 **같은 프로젝트**에서 쓸 때
- **코드 변경 없이** 토큰을 절약하고 싶을 때 (프록시 모드)
- RAG 시스템에서 **대량 컨텍스트**를 LLM에 전달할 때
- 긴 로그/툴 출력에서 **정확도 유지**하면서 토큰 줄이고 싶을 때

### 스킵할 상황 👎
- 단일 공급자의 네이티브 compaction만 사용하고 크로스에이전트 메모리가 필요 없을 때
- 샌드박스 환경에서 로컬 프로세스를 실행할 수 없을 때
- 컨텍스트가 항상 작아서(수백 토큰) 압축 이점이 없을 때

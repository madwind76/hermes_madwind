# Weaponization (무기화) — 사이버 킬 체인 2단계

## Step 1: 단어 직역 및 쉬운 비유

### 1. 약자 풀이

**Weaponization** = **Weapon** (무기) + **-ization** (~화, 만들다)

| 영어 단어 | 직역 | 의미 |
|-----------|------|------|
| **Weapon** | 무기, 병기 | 공격에 사용하는 도구 |
| **-ization** | ~화, 만들다 | 어떤 상태로 만드는 과정 |

### 2. 의미 조합

> **"공격 도구(무기)를 만들어내는 과정 — 취약점을 악용하는 익스플로잇 코드와 악성 페이로드를 결합하여 전달 가능한 악성 파일/스크립트 제작"**

### 3. 강력한 비유: "폭탄 조립 공장의 무기 설계도"

```
┌────────────────────────────────────────────────────────────┐
│  상황: 테러리스트(공격자)가 목표 건물을 폭파하기 위해      │
│  맞춤형 폭탄(악성코드)을 조립하는 과정                     │
└────────────────────────────────────────────────────────────┘

1️⃣  **설계도(익스플로잇 코드)**: "이 건물의 3층 창문 잠금장치는 
    열쇠 없이도 특정 충격으로 열린다" → **취약점 악용 코드**
    
2️⃣  **폭발물(페이로드)**: 실제로 터져서 피해를 주는 부분 
    → **랜섬웨어 암호화 루틴, 백도어, 키로거, 데이터 탈취 스크립트**

3️⃣  **폭탄 케이스(드로퍼/래퍼)**: 폭발물을 감싸서 목표물까지 
    무사히 가져가는 포장 → **매크로 문서, PDF, 설치자, 스크립트**

4️⃣  **타이머/기폭장치(트리거)**: 특정 조건(시간, 사용자 행동)에서 
    작동 → **자동 실행 매크로, 사용자 클릭, 특정 날짜/시간**

💡 **핵심 포인트**: 정찰에서 알아낸 **"어떤 창문이 열리는지(취약점)"** 정보를 바탕으로, 
**"그 창문으로 들어가서 무슨 짓을 할지(페이로드)"** 를 담은 **완성된 무기(악성 파일)** 를 만듭니다. 
아직 던지진 않았고(Delivery 안 함), **내부에서 조립만 완료된 상태**입니다.
```

## Step 2: 개념 시각화

![Weaponization 비유 시각화: 공격자 작업장에서 악성코드 조립 — 익스플로잇 코드(설계도), 페이로드(폭발물), 백도어(숨겨진 공간), 드로퍼(전달 장치), 완성된 악성 파일 - 한글 레이블 포함](https://v3b.fal.media/files/b/0a9dfbcc/qluM8gaL4T3redTsZJ8be_DsTvZVST.png)

**이미지 설명**:
- **공격자(해커)** — 무기화 작업을 수행하는 주체
- **무기화 작업장** — 공격자 인프라(개발 환경, 빌드 시스템)
- **익스플로잇 코드(설계도)** — 취약점 악용 로직 (ROP 체인, 쉘코드 등)
- **페이로드(폭발물)** — 실제 악성 행위 수행 코드 (랜섬웨어, 백도어, 스파이웨어)
- **백도어(숨겨진 공간)** — 지속성 확보를 위한 은밀한 접근 경로
- **드로퍼(전달 장치)** — 페이로드를 목표 시스템에 드롭하는 로더
- **완성된 악성 파일** — 전달 단계에서 사용할 최종 무기 (문서, 실행파일, 스크립트)

## Step 3: 전문 용어 설명 (위키백과 기반)

### 무기화 (Weaponization, 사이버 킬 체인)

**정의**: **사이버 킬 체인의 2단계**로, 공격자가 1단계 정찰에서 수집한 대상 정보(취약점, 소프트웨어 버전, 방어 체계 등)를 바탕으로 **실제 공격에 사용할 악성코드(무기)를 개발·조립·패키징하는 과정**이다.

### 무기화의 핵심 구성요소

| 구성요소 | 역할 | 예시 |
|----------|------|------|
| **익스플로잇 (Exploit)** | 취약점을 악용해 코드 실행 권한 획득 | CVE-2021-44228 Log4j JNDI 인젝션, EternalBlue SMB 익스플로잇 |
| **페이로드 (Payload)** | 권한 획득 후 수행할 실제 악성 동작 | 랜섬웨어 암호화, 리버스 쉘, 키로거, 데이터 탈취, 크립토마이너 |
| **드로퍼/다운로더 (Dropper/Downloader)** | 페이로드를 시스템에 설치/다운로드 | 매크로 문서, 악성 설치자, PowerShell 스크립트, HTA 파일 |
| **페커/크라이터 (Packer/Crypter)** | 시그니처 회피를 위한 난독화/압축 | UPX, VMProtect, 커스텀 인코더 |
| **지속성 메커니즘** | 재부팅/로그아웃 후에도 생존 | 레지스트리 Run 키, 스케줄드 태스크, 서비스, WMI 이벤트 구독 |

### 무기화 기법 유형

| 유형 | 설명 | 대표 사례 |
|------|------|-----------|
| **문서 기반** | Office 매크로, PDF 취약점, OLE 임베딩 | APT28 CHOPSTICK, TA505 Dridex 매크로 |
| **스크립트 기반** | PowerShell, JavaScript, VBScript, HTA | fileless 공격, Living-off-the-Land |
| **실행 파일** | PE/ELF 바이너리, 쉘코드 로더 | Cobalt Strike Beacon, Metasploit Meterpreter |
| **공급망** | 정식 소프트웨어 빌드 과정에 악성코드 삽입 | SolarWinds SUNBURST, CodeCov, 3CX |
| **컨테이너/이미지** | Docker 이미지, Kubernetes 매니페스트 악성코드 | 악성 베이스 이미지, 공급망 공격 |

### 무기화 단계에서의 공격자 고려사항

| 고려사항 | 설명 |
|----------|------|
| **타겟 환경 적합성** | OS 버전, 아키텍처(x86/x64/ARM), 설치된 소프트웨어, AV/EDR 종류 |
| **탐지 회피** | 시그니처 기반 탐지 우회(패킹, 암호화), 행위 기반 탐지 우회(지연 실행, 정상 프로세스 위장) |
| **신뢰성** | 다양한 환경에서 안정적 동작, 크래시 방지, 샌드박스 탐지(안티 샌드박스) |
| **모듈성** | 페이로드 교체 용이, C2 서버 변경 용이, 플러그인 아키텍처 |
| **작전 보안(OPSEC)** | 아티팩트 최소화, 디버그 심볼 제거, 컴파일 타임스탬프 조작, 고유 식별자 제거 |

### 방어 관점: 무기화 탐지 및 차단

| 방어 계층 | 대응 방안 |
|----------|-----------|
| **엔드포인트** | EDR 행위 분석(매크로 실행, PowerShell 의심 패턴, 프로세스 인젝션), AMSI(안티멀웨어 스캔 인터페이스) |
| **네트워크** | 샌드박스 자동 분석(파일/URL 평판), CDR(Content Disarm & Reconstruction)로 문서 내 매크로/활성 콘텐츠 제거 |
| **이메일/웹** | 첨부파일 차단/격리(실행파일, 매크로 문서), URL 평판 검사, 샌드박스 폭파 |
| **애플리케이션** | 매크로 기본 차단, Protected View, 애플리케이션 화이트리스트(AppLocker, WDAC) |
| **위협 인텔리전스** | 알려진 무기화 아티팩트(해시, IP, 도메인, YARA 룰) 공유, 신규 익스플로잇 시그니처 신속 배포 |

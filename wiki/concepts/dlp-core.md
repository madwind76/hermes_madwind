## Step 1: 단어 직역 및 쉬운 비유

### 1. 약자 풀이

**DLP** = **D**ata **L**oss **P**revention

| 약자 | 원래 단어 | 직역 | 의미 |
|------|-----------|------|------|
| **D** | **Data** | 데이터, 정보 | 디지털화된 지식/자산 |
| **L** | **Loss** | 손실, 유실 | 의도치 않게 빠져나감 |
| **P** | **Prevention** | 예방, 방지 | 미리 막는 조치 |

### 2. 의미 조합

> **"중요 데이터(개인정보, 기밀문서, 지적재산)가 조직 밖으로 유출되는 것을 **사전에 탐지하고 차단**하는 기술·정책·프로세스의 통합 체계"**

### 3. 강력한 비유: "회사 출구마다 설치된 **스마트 세관 검색대**"

```
┌────────────────────────────────────────────────────────────┐
│  상황: 직원들(데이터)이 회사 건물(네트워크/엔드포인트/클라우드) │
│  을 나서려 함. 기존 경비(방화벽/권한)는 "신분증(접근권한)"만  │
│  확인하고 통과시킴. DLP는 "짐 속 내용물까지 검사" 함        │
└────────────────────────────────────────────────────────────┘

🛃  **스마트 세관의 4단계 검사 프로세스**

  ① **무엇을 가지고 나가는가? (데이터 식별/Classification)**
      - "이 가방엔 **주민등록번호 1,000개**, 저 서류엔 **설계도면 PDF**" 
      - 정규식(Regex): 신용카드 번호 `^\d{4}[- ]?\d{4}[- ]?\d{4}[- ]?\d{4}$`
      - 키워드/사전: "기밀", "대외비", 프로젝트 코드명
      - 지문(Fingerprinting): 원본 문서 해시 등록 → 변형/일부 복사도 탐지
      - 머신러닝: 문서 구조/메타데이터/언어 모델로 민감도 자동 판별

  ② **어디로 나가려는가? (채널 모니터링/Channel Monitoring)**
      - 📧 **이메일**: 첨부파일, 본문, 전달/회신/BCC
      - ☁️ **클라우드/웹**: Google Drive, Dropbox, GitHub, Slack, Teams, 웹메일 업로드
      - 💾 **이동식 매체**: USB, 외장하드, SD카드 (포트 제어 연동)
      - 🖨️ **인쇄/팩스**: 출력 내용 OCR 스캔, 워터마크 강제
      - 💬 **메신저/협업툴**: 카카오톡, 라인, 디스코드, 노션, 컨플루언스

  ③ **누가, 왜, 어떻게? (컨텍스트 분석/Context Analysis)**
      - **사용자**: 부서, 직급, 퇴사 예정자(HR 연동), 고위험군
      - **행위**: 대량 다운로드, 야간/주말 접속, 비정상 경로, 압축/암호화 시도
      - **데이터 상태**: 정형(DB/CSV), 비정형(문서/이미지), 코드/스크립트

  ④ **어떻게 대응할까? (정책 기반 액션/Policy Enforcement)**
      - 🚫 **차단(Block)**: 전송 중단, "기밀정보 포함되어 전송 불가" 알림
      - ⚠️ **경고/승인(Alert/Justify)**: "정말 보내시겠습니까? 사유 입력" → 관리자 승인 대기
      - 📝 **암호화/워터마크(Encrypt/Mark)**: 강제 암호화, 사용자 ID/시간 워터마크 삽입
      - 📋 **격리/로깅(Quarantine/Log)**: 의심 파일 샌드박스 격리, 포렌식용 원본 보관
      - 🧹 **자동 삭제/마스킹(Redact)**: 이메일 본문에서 주민번호 `******` 마스킹

💡 **핵심 포인트**: 방화벽이 **"문(포트/프로토콜)을 지킴"**, 
DLP는 **"문으로 나가는 짐(데이터 내용물)을 검사함"**.
**"데이터가 어디에 있든(At Rest/In Motion/In Use), 무엇이 민감한지 알고, 나가는 길목마다 지킴"**
```

## Step 3: 전문 용어 설명 (위키백과 기반)
### DLP (Data Loss Prevention, 데이터 유출 방지)

**정의**: **DLP(Data Loss Prevention)**는 조직의 **민감 데이터(개인식별정보 PII, 결제카드정보 PCI, 건강정보 PHI, 지적재산 IP, 영업비밀, 기밀문서 등)가 의도적/비의도적으로 외부로 유출되는 것을 탐지·모니터링·차단하기 위한 기술 솔루션, 정책, 프로세스의 통합 체계**이다. "데이터 누수 방지(Data Leak Prevention)"라고도 불린다.

### DLP의 3대 보호 상태 (Data States)

| 상태 | 설명 | DLP 적용 지점 |
|------|------|---------------|
| **Data at Rest (저장 중)** | DB, 파일 서버, 스토리지, 백업, 아카이브, 쉐어포인트, NAS | 스토리지 스캔, 파일 서버 에이전트, DB 활동 모니터링(DAM), 클라우드 스토리지 API 스캔 |
| **Data in Motion (전송 중)** | 네트워크 트래픽(이메일, 웹, FTP, P2P, VPN, 메시징) | 네트워크 DLP(미러 포트/프록시), 이메일 게이트웨이, 웹 프록시/CASB, API 게이트웨이 |
| **Data in Use (사용 중)** | 엔드포인트에서 열람/편집/복사/인쇄/화면 캡처/클립보드 | 엔드포인트 에이전트(커널 후킹), 화면 워터마크, 인쇄 제어, 클립보드 모니터링, 가상 데스크톱(VDI) 연동 |

### 주요 탐지 기법 (Detection Techniques)

| 기법 | 원리 | 장점 | 한계 | 보완 |
|------|------|------|------|------|
| **정규식/패턴 매칭 (Regex)** | 정의된 패턴(주민번호, 카드번호, 이메일, IP) 매칭 | 구현 간단, 빠름, 거짓 양성 낮음(패턴 명확시) | 변형/인코딩/이미지 내 텍스트 탐지 불가 | OCR, 인코딩 디코딩, 이미지 분석 연계 |
| **키워드/사전 매칭** | 금지어/프로젝트명/분류 라벨 사전 기반 | 비정형 문서(기밀, 비밀, 프로젝트명) 탐지 | 문맥 무시 → 과탐(오탐) 많음 | 근접도 분석, 가중치 스코어링, ML 분류 |
| **데이터 지문 (Fingerprinting/Exact Match)** | 원본 문서 해시(청크 단위) DB화 → 유사도 비교 | 버전 변경/일부 복사/포맷 변환에도 탐지 | 원본 등록 필요, 대용량 DB 성능 이슈 | 증분 인덱싱, 빌딩 블록(청크) 해시 |
| **통계/구조 분석 (Statistical/Structural)** | 문서 구조(헤더/푸터/표/양식), 메타데이터, 엔트로피 | 템플릿 기반 문서(신청서, 보고서) 탐지 | 자유 형식 문서엔 약함 | ML 분류기와 앙상블 |
| **머신러닝/딥러닝 (ML/DL)** | 문서 임베딩(BERT, TF-IDF 등) → 민감도 분류, 이상 탐지 | 문맥 이해, 변형 강건, 신규 유형 일반화 | 학습 데이터 필요, 설명성 낮음, 연산 비용 | 앙상블, 경량화(Distillation), 온라인 학습 |
| **광학 문자 인식 (OCR)** | 이미지/PDF 스캔본/스크린샷 내 텍스트 추출 → 분석 | 이미지 유출(사진 촬영, 스캔본) 탐지 | 해상도/언어/레이아웃 의존 | 전처리(이미지 보정), 다국어 모델 |
| **워터마크/스테가노그래피 분석** | 비가시 워터마크 임베디드/추출, 숨겨진 데이터 탐지 | 인쇄물/스크린샷 추적, 내부자 식별 | 삽입/검증 인프라 필요 | 출력/화면 워터마크 솔루션 연계 |

### 채널별 DLP 아키텍처

| 채널 | 구축 방식 | 주요 기능 | 대표 솔루션 연계 |
|------|-----------|-----------|------------------|
| **네트워크 DLP** | 미러 포트(SPAN/TAP), 인라인 프록시, 전용 어플라이언스 | 전사 트래픽 가시성, 이메일/SMTP/HTTP/FTP/DNS/IMAP/POP3 분석 | 방화벽, 프록시, 이메일 게이트웨이, CASB |
| **엔드포인트 DLP** | 커널 드라이버/미니필터/ETW/eBPF 에이전트 | 파일 열람/복사/이동/업로드/인쇄/클립보드/화면 캡처 제어 | EDR, EPP, MDM/EMM, VDI |
| **이메일 DLP** | MTA 연동(Postfix/Exchange/Office 365/Gmail API), ICAP | 첨부/본문/제목/수신자 검사, 암호화/차단/승인 워크플로 | 이메일 보안 게이트웨이(SEG), Microsoft Purview, Google Vault |
| **클라우드/CASB DLP** | API 모드(구글/MS/박스/세일즈포스/슬랙/깃허브), 프록시 모드 | SaaS 앱 내 파일/메시지/댓글 스캔, 공유 설정/권한 감사 | CASB(Netskope, McAfee MVISION, Microsoft Defender for Cloud Apps) |
| **데이터베이스 DLP (DAM)** | DB 프록시/에이전트/로그 분석 | SQL 쿼리/결과셋 모니터링, 대량 조회/내보내기 탐지, 마스킹 | DB 방화벽(Imperva, FortiDB, Oracle Audit Vault) |
| **인쇄/팩스 DLP** | 프린트 스풀러 후킹, MFP(복합기) 임베디드, 팩스 서버 연동 | 출력 내용 OCR/텍스트 추출, 워터마크 강제, 출력 로그 | 출력 관리 솔루션(Papercut, SafeCom, YSoft) |

### 정책 엔진 및 대응 액션 (Policy Engine & Actions)

| 액션 유형 | 설명 | 적용 예시 |
|----------|------|-----------|
| **차단 (Block/Prevent)** | 전송/저장/복사/인쇄 즉시 중단, 사용자 알림 | USB 복사 차단, 이메일 첨부 차단, 클라우드 업로드 차단 |
| **경고/사유 입력 (Warn/Justify)** | 사용자에게 위험 알림, 비즈니스 사유 입력 강제, 관리자 승인 대기 | "기밀 문서입니다. 전송 사유를 입력하세요" → 승인 후 전송 |
| **강제 암호화 (Encrypt)** | 파일/첨부/이메일 본문 자동 암호화(AES-256), 비밀번호 별도 통보 | 외부로 나가는 기밀 보고서 자동 암호화 |
| **워터마크 삽입 (Watermark)** | 사용자 ID, 시간, 세션 ID, 문서 분류 라벨 비가시/가시 삽입 | 출력물/PDF/이미지에 "CONFIDENTIAL - 홍길동 - 2026-06-12" |
| **마스킹/리댁션 (Mask/Redact)** | 민감 필드(주민번호, 카드번호) `****` 또는 토큰화 치환 | 이메일 본문에서 카드번호 `****-****-****-1234` 치환 |
| **격리/샌드박스 (Quarantine)** | 의심 파일 별도 저장소 격리, 관리자 검토 후 해제/삭제 | 알려지지 않은 실행파일, 매크로 문서 격리 |
| **로깅/포렌식 보존 (Log/Archive)** | 원본 파일/메타데이터/컨텍스트(유저/프로세스/시간) 장기 보관 | 감사/컴플라이언스/사고 조사용 (WORM 스토리지) |
| **자동 분류/라벨링 (Classify/Label)** | Microsoft Sensitivity Labels(AIP), Google Drive Labels 연동 | "기밀/대외비/공개" 라벨 자동 부여 → 하위 정책 상속 |

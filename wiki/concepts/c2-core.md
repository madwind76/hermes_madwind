# Command & Control (C2, 명령 및 제어) — 사이버 킬 체인 6단계

## 참고 URL
- [ko.wikipedia.org](https://ko.wikipedia.org/wiki/사이버_킬_체인)

## Step 1: 단어 직역 및 쉬운 비유

### 1. 약자 풀이

**C2** = **C**ommand **&** **C**ontrol

| 약자 | 원래 단어 | 직역 | 의미 |
|------|-----------|------|------|
| **C** | **Command** | 명령, 지시 | 공격자가 내리는 작업 지시 |
| **&** | **and** | 그리고 | 연결 |
| **C** | **Control** | 제어, 통제 | 대상 시스템을 원격 조종 |

**Command and Control** = **명령 및 제어** / **C2 서버** / **C&C 서버**

### 2. 의미 조합

> **"설치된 임플란트(백도어/비콘)가 공격자 서버와 통신하며 명령을 수신하고 결과를 전송하는 채널 — 공격자가 원격에서 침투 시스템을 조종하는 '무선 리모컨'"**

### 3. 강력한 비유: "인형극 조종사가 끈으로 마리오네트 조종하는 무대"

```
┌────────────────────────────────────────────────────────────┐
│  상황: 건물 곳곳에 심어둔 도청기/원격 제어 장치(임플란트)  │
│  공격자 기지국(C2 서버)에서 실시간으로 조종                │
└────────────────────────────────────────────────────────────┘

🎮  **원격 조종실(C2 인프라)** — 어떻게 인형들을 조종할까?

  ① **무선 리모컨(비콘/임플란트)**: 각 인형(감염 시스템)에 
      달린 수신기 → 주기적으로 "명령 있어?" 묻고(Beaconing), 
      지시 받으면 실행 후 결과 보고

  ② **보이지 않는 끈(C2 채널/프로토콜)**: 
      - **HTTPS/웹 소켓**: 정상 웹 트래픽인 척 위장 (가장 흔함)
      - **DNS 터널링**: DNS 쿼리/응답에 데이터 숨김 (방화벽 우회)
      - **커스텀 TCP/UDP**: 독자 프로토콜, 포트 임의 지정
      - **소셜 미디어/클라우드**: GitHub/Gist, Twitter, Telegram, Discord, 
        Google Drive, AWS S3 API 악용 (도메인 프론팅)
      - **P2P/메시**: 중앙 서버 없이 감염 노드 간 직접 통신 (단일 장애점 제거)

  ③ **조종 콘솔(C2 프레임워크/패널)**: 
      - **상용/오픈소스**: Cobalt Strike, Sliver, Mythic, Havoc, Brute Ratel, 
        Metasploit, Empire, Koadic, Shad0w
      - **기능**: 비컨 관리, 태스크 큐, 모듈 로드(파일 업/다운, 스크린샷, 
        키로거, 자격증명 덤프, 수평 이동), 레포팅

  ④ **작전 보안(OPSEC)**: 
      - **도메인 프론팅/ CDN**: Cloudflare, Akamai 뒤 숨기기
      - **리다이렉터/프록시 체인**: 트래픽 세탁, 실제 C2 IP 은폐
      - **비콘 지터/슬립**: 불규칙 간격으로 통신, 탐지 회피
      - **암호화/인코딩**: TLS, AES, XOR, Base64, 커스텀 인코더
      - **킬 스위치**: 작전 중단 시 전체 임플란트 자폭

💡 **핵심 포인트**: 설치가 **"비밀 통로 만들기"**라면, C2는 **"그 통로로 실시간 명령 내리기"**입니다. 
**비콘(Beaconing)** — 주기적 생존 신호 + 명령 폴링이 C2의 심장박동입니다.
```

## Step 2: 개념 시각화

![C2 비유 시각화: 인형극 조종사로 설명하는 명령 및 제어 — 공격자(해커), C2 서버(제어판), 암호화된 C2 채널(끈), 심어진 에이전트(인형), 대상 네트워크(무대), 태스크 큐, 비컨 상태, 모듈 로딩, 데이터 유출 - 한글 레이블 포함](https://v3b.fal.media/files/b/0a9dfc64/Vvc4G458wT-pNeAjcu1r5_YmOJcVmj.png)

**이미지 설명**:
- **공격자(해커)** — C2 콘솔에서 작전을 지휘하는 주체
- **C2 서버(제어판)** — 명령 전달, 비컨 관리, 데이터 수집 중앙 서버
- **암호화된 C2 채널(끈)** — HTTPS, DNS, WebSocket, 커스텀 프로토콜 등 은폐된 통신 경로
- **심어진 에이전트(인형)** — 설치 단계에서 심어진 비콘/임플란트 (비컨: 주기적 생존 신호)
- **대상 네트워크(무대)** — 침투된 조직 내부 네트워크
- **태스크 큐** — 공격자가 예약한 작업 목록 (파일 수집, 스크린샷, 수평 이동 등)
- **비컨 상태** — 온라인/오프라인, 마지막 체크인, 버전, 권한 수준
- **모듈 로딩** — 런타임에 기능 확장 (Reflective DLL Injection, Assembly Load)
- **데이터 유출** — 수집된 파일/자격증명/스크린샷이 C2로 전송됨

## Step 3: 전문 용어 설명 (위키백과 기반)
### 명령 및 제어 (Command and Control, C2, 사이버 킬 체인)

**정의**: **사이버 킬 체인의 6단계**로, 공격자가 대상 시스템에 설치한 임플란트(비콘, 백도어, 에이전트)와 **암호화된 은밀한 통신 채널을 통해 원격으로 명령을 전달하고, 실행 결과를 수신하며, 수집된 데이터를 유출하는 인프라 및 프로토콜**이다. 공격자가 **침투 환경을 실시간으로 조종**하는 핵심 단계다.

### C2 아키텍처 구성요소

| 구성요소 | 역할 | 상세 |
|----------|------|------|
| **C2 서버 (Team Server)** | 중앙 지휘부 | 비컨 관리, 태스크 분배, 데이터 수집, 오퍼레이터 콘솔 제공 |
| **리다이렉터/프록시** | 트래픽 세탁 | 실제 C2 IP 은폐, 도메인 프론팅, CDN 악용, 레이어드 프록시 체인 |
| **비컨/임플란트 (Agent)** | 감염 호스트 상주 | 주기적 체크인(Beaconing), 태스크 수신/실행/결과 전송, 모듈 로드 |
| **C2 채널/프로토콜** | 통신 경로 | HTTPS, DNS, WebSocket, gRPC, QUIC, 커스텀 TCP/UDP, P2P |
| **페이로드/모듈** | 기능 확장 | 파일 업/다운, 셸, 스크린샷, 키로거, 자격증명 덤프, 수평 이동, 스캐너 |

### 주요 C2 통신 프로토콜 및 기법

| 프로토콜/기법 | 특징 | 장점 | 탐지 난이도 |
|--------------|------|------|-------------|
| **HTTPS (HTTP/2, HTTP/3)** | 정상 웹 트래픽 모방, TLS 암호화 | 가장 널리 쓰임, 프록시/방화벽 통과 용이 | 중간 (JA3 지문, SNI, 인증서 분석) |
| **DNS 터널링** | DNS 쿼리/응답에 데이터 인코딩 | 아웃바운드 DNS 거의 허용, 포트 53 개방 | 중간 (엔트로피 분석, 쿼리 빈도/크기) |
| **WebSocket / gRPC** | 양방향 스트리밍, 낮은 레이턴시 | 실시간 인터랙티브 쉘, 양방향 통신 | 중간 (프로토콜 분석, 핸드셰이크) |
| **소셜 미디어/클라우드 API** | GitHub, Twitter, Telegram, Discord, GDrive, S3 | 합법적 도메인, 차단 어려움, 인증 내장 | 높음 (행위 분석, API 호출 패턴) |
| **도메인 프론팅** | CDN(SNI=정상, Host=악성) 뒤 숨김 | 차단 회피, 실제 C2 도메인 은폐 | 높음 (CDN 업체 차단 추세) |
| **P2P / 메시 네트워크** | 중앙 서버 없이 노드 간 직접 통신 | 단일 장애점 제거, 테익다운 어려움 | 높음 (네트워크 토폴로지 분석 필요) |
| **커스텀 바이너리 프로토콜** | 독자 포맷, 시그니처 없음 | 시그니처 기반 탐지 회피 | 높음 (행위/머신러닝 기반 필요) |

### 비콘(Beaconing) 동작 원리

```
┌─────────────────────────────────────────────────────────────┐
│                    비콘 라이프사이클                          │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  [임플란트 실행]                                              │
│        │                                                      │
│        ▼                                                      │
│  [초기 체크인] ──→ C2 서버에 등록 (시스템 정보, 권한, 버전)   │
│        │                                                      │
│        ▼                                                      │
│  ┌─────────────────────────────────────────┐                 │
│  │           비콘 루프 (무한 반복)           │                 │
│  ├─────────────────────────────────────────┤                 │
│  │  1. 슬립 (Sleep/Jitter: 30초~수시간,    │                 │
│  │     랜덤 지터로 패턴 회피)               │                 │
│  │        │                                │                 │
│  │        ▼                                │                 │
│  │  2. HTTP GET/POST → C2 (비컨 전송)      │                 │
│  │     - 메타데이터: 호스트명, 사용자, PID, │                 │
│  │       권한, OS, 마지막 태스크 결과       │                 │
│  │        │                                │                 │
│  │        ▼                                │                 │
│  │  3. 태스크 수신 (JSON/직렬화 바이너리)  │                 │
│  │     - 태스크 ID, 타입, 파라미터         │                 │
│  │        │                                │                 │
│  │        ▼                                │                 │
│  │  4. 태스크 실행 (모듈 로드/쉘/명령)     │                 │
│  │        │                                │                 │
│  │        ▼                                │                 │
│  │  5. 결과 전송 (POST, 청크 분할 가능)    │                 │
│  └─────────────────────────────────────────┘                 │
│        │                                                      │
│        ▼ (반복)                                               │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

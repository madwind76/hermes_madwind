# File Upload (파일 업로드 취약점, 제한 없는 파일 업로드) — 보안 용어 해설

## Step 1: 단어 직역 및 쉬운 비유

### 1. 용어 풀이

**File Upload Vulnerability** = **File** (파일) + **Upload** (업로드, 올리기) + **Vulnerability** (취약점, 약점)

| 영어 단어 | 직역 | 의미 |
|-----------|------|------|
| **File** | 파일, 문서 | 데이터 묶음 (이미지, 문서, 실행파일 등) |
| **Upload** | 업로드, 올리기 | 클라이언트 → 서버로 파일 전송 |
| **Vulnerability** | 취약점, 약점 | 악용 가능한 보안 결함 |

**Unrestricted File Upload** = **Unrestricted** (제한 없는) + **File Upload** = **파일 형식/크기/내용 검증 없이 아무 파일이나 올릴 수 있는 상태**

### 2. 의미 조합

> **"웹 애플리케이션이 사용자 파일 업로드 시 파일 형식, 내용, 크기 등을 제대로 검증하지 않아, 공격자가 악성 파일(웹쉘, 악성코드, 스크립트)을 업로드하여 서버에서 실행 가능하게 만드는 취약점"**

### 3. 강력한 비유: "검문소 없는 창고에 폭탄 상자 맡기기"

```
┌────────────────────────────────────────────────────────────┐
│  상황: 회사 창고(서버)에 누구나 택배(파일)를 맡길 수 있음   │
│  검문소(검증)가 없어서 폭탄(웹쉘)도 그냥 받아줌             │
└────────────────────────────────────────────────────────────┘

📦  **무방비 창고 시나리오 (파일 업로드 공격 흐름)**

  ① **정상 이용**: 직원(사용자)이 서류(이미지/문서) 맡김
     - 창고지기(서버): "어, 이미지네? 받아줘." → 창고에 저장

  ② **공격자 악용**: 해커가 폭탄(악성 파일) 상자 맡김
     - 상자 겉에 "이미지.jpg" 라벨 붙임 (확장자 위장)
     - 안에는 웹쉘.php (PHP 코드) 또는 악성 스크립트 들어있음
     - Content-Type: image/jpeg 로 위장 (MIME 타입 속임)

  ③ **창고지기의 순진한 수락**: 
     - "확장자가 .jpg네? 이미지 맞네? 받아줘!"
     - 내부 내용물 확인 안 함 (매직 바이트/파일 시그니처 미확인)
     - 서버 웹 루트(/var/www/uploads/)에 그대로 저장

  ④ **폭탄 터뜨리기 (실행)**: 
     - 공격자: `https://site.com/uploads/shell.php` 접속
     - 서버: "어? PHP 파일이네? 실행해줘야지!" → 웹쉘 실행
     - 결과: 서버 완전 장악 (파일 읽기/쓰기, 명령 실행, DB 접근 등)

💡 **핵심 포인트**: 
- **"확장자만 보고 믿지 마라"** — 확장자/MIME 타입은 클라이언트가 조작 가능
- **파일 내용(시그니처/매직 바이트) 확인 필수** — 진짜 이미지인지 확인
- **업로드 디렉토리 실행 권한 제거 필수** — 업로드 폴더에서 PHP/스크립트 실행 차단
- **"저장만 하고 실행 안 하면 안전하다"는 착각 금물** — 경로 탐색(Path Traversal) 등으로 실행 가능
```

## Step 2: 개념 시각화

![File Upload 비유 시각화: 검문소 없는 창고로 설명하는 파일 업로드 — 사용자(택배 기사), 창고(서버 업로드 디렉토리), 정상 파일(서류 상자), 위장된 악성 파일(폭탄 상자), 검문소 없음(검증 누락), 웹쉘 실행(폭탄 터짐) - 한글 레이블 포함](https://v3b.fal.media/files/b/0a9dfef4/KqR2mKxL5vN8tYpHgJkB4_L9wEmVnA.png)

**이미지 설명**:
- **사용자(택배 기사)** — 파일을 업로드하는 클라이언트 (정상 사용자 또는 공격자)
- **창고(서버 업로드 디렉토리)** — 파일 저장소, 웹에서 접근 가능한 경로
- **정상 파일(서류 상자)** — 이미지, PDF, 문서 등 허용된 파일
- **위장된 악성 파일(폭탄 상자)** — 확장자/MIME 위장한 웹쉘, 악성 스크립트
- **검문소 없음(검증 누락)** — 확장자/MIME/내용/크기 검증 누락
- **웹쉘 실행(폭탄 터짐)** — 업로드된 파일이 서버에서 코드로 실행됨

> ⚠️ **참고**: 이미지 생성 도구가 PNG 형식으로 반환했습니다. 스킬 요구사항(.jpg/.jpeg)은 현재 도구 제약상 PNG로 대체됩니다.

## Step 3: 전문 용어 설명 (위키백과/OWASP/PortSwigger 기반)
### 파일 업로드 취약점 (File Upload Vulnerability, Unrestricted File Upload)

**정의**: **파일 업로드 취약점**은 웹 애플리케이션이 **사용자가 업로드한 파일의 형식(확장자, MIME 타입), 내용(매직 바이트, 시그니처), 크기, 이름 등을 충분하게 검증하지 않아**, 공격자가 **악성 파일(웹쉘, 악성코드, 스크립트, 악성 문서)을 업로드하고 서버에서 실행하거나 다운로드하게 만드는 취약점**이다. OWASP Top 10에서 관련 카테고리로 지속적으로 언급된다.

### 공격 메커니즘: 왜 실행되는가?

| 조건 | 설명 |
|------|------|
| **업로드 디렉토리 웹 접근 가능** | `/uploads/`, `/assets/`, `/files/` 등이 웹에서 직접 접근 가능 (DocumentRoot 하위) |
| **스크립트 실행 권한 있음** | 업로드 디렉토리에서 PHP, ASP, JSP, Python, Node.js 등 인터프리터 실행 가능 |
| **파일 확장자/MIME 검증만 수행** | 클라이언트 측 조작 가능한 값만 검증, 서버 측 내용 검증 없음 |
| **파일 이름/경로 제어 가능** | 공격자가 파일명/경로 제어 → `shell.php.jpg` → `shell.php` 우회 등 |

### 주요 공격 기법 및 우회 방법

| 기법 | 설명 | 우회 대상 |
|------|------|-----------|
| **이중 확장자** | `shell.php.jpg`, `shell.php.png`, `shell.php%00.jpg` | 확장자 블랙리스트/화이트리스트 |
| **대소문자 변형** | `shell.PHp`, `shell.pHp`, `shell.phP` | 대소문자 구분 필터 |
| **NULL 바이트 인젝션** | `shell.php%00.jpg` (구버전 PHP/Apache) | 확장자 검증 로직 (문자열 종료 악용) |
| **MIME 타입 위장** | `Content-Type: image/jpeg` 전송하며 실제로는 PHP | MIME 타입만 검증 시 |
| **매직 바이트 위장** | JPEG 헤더(`FF D8 FF`) + PHP 코드 결합 (`FF D8 FF ... <?php system($_GET['cmd']); ?>`) | 파일 시그니처(매직 바이트)만 검증 시 |
| **확장자 파싱 차이** | `shell.php.xxx` → Apache `AddHandler` 우선순위 악용 | 웹서버 확장자 파싱 순서 |
| **경로 탐색 결합** | `../../../var/www/shell.php` (Path Traversal) | 업로드 경로 제어 |
| **압축 파일 업로드** | ZIP/TAR/GZ 내 웹쉘 → 서버 측 압축 해제 기능 악용 | 압축 해제 기능 |
| **이미지 메타데이터** | EXIF/XMP/댓글 필드에 PHP 코드 삽입 → `exif_read_data()` 등 악용 | 이미지 처리 라이브러리 |
| **SVG/XML 업로드** | `<svg onload="alert(1)"/>` (XSS) 또는 XXE 페이로드 | SVG 렌더링/파싱 |
| **파일명 인코딩** | URL 인코딩, 유니코드, NTFS ADS(`shell.php::$DATA`) | 파일시스템/파서 차이 |

### 파일 형식별 위험도

| 파일 형식 | 위험도 | 주요 위험 |
|----------|--------|-----------|
| **스크립트/실행파일** (`.php`, `.asp`, `.aspx`, `.jsp`, `.py`, `.pl`, `.sh`, `.exe`, `.jar`) | **치명적** | 직접 코드 실행 → 웹쉘, RCE |
| **웹 템플릿** (`.twig`, `.jade`, `.ejs`, `.erb`, `.volt`) | **높음** | 템플릿 엔진 SSTI → RCE |
| **설정/직렬화** (`.yaml`, `.yml`, `.json`, `.xml`, `.config`, `.ini`) | **높음** | 역직렬화/RCE, XXE, 설정 조작 |
| **문서/오피스** (`.docx`, `.xlsx`, `.pptx`, `.odt`, `.pdf`) | **중간** | 매크로/VBA, 임베디드 객체, XXE, PDF JS 실행 |
| **이미지** (`.jpg`, `.png`, `.gif`, `.bmp`, `.svg`, `.webp`) | **낮음~중간** | 매직 바이트 위장, SVG XSS/XXE, EXIF 코드 실행 |
| **아카이브** (`.zip`, `.tar`, `.gz`, `.rar`, `.7z`) | **중간** | 압축 해제 시 경로 탐색, 심볼릭 링크, 내부 웹쉘 |
| **폰트/미디어** (`.woff`, `.ttf`, `.mp4`, `.mp3`) | **낮음** | 파서 취약점(CVE) 악용 가능성 |

### 파일 업로드 방어 기법 (다층 방어 - Defense in Depth)

| 방어 계층 | 기법 | 구현 예시 | 효과/비고 |
|----------|------|-----------|-----------|
| **1. 클라이언트 측 (보조)** | **확장자/MIME 화이트리스트** | `accept="image/png,image/jpeg,application/pdf"` | UX용, 우회 가능 → **절대 단독 신뢰 금지** |
| | **파일 크기 제한** | `max-file-size: 5MB` (클라이언트 + 서버) | DoS/대용량 악성 파일 1차 차단 |
| **2. 서버 측 검증 (핵심)** | **확장자 화이트리스트** | `allowed_ext = {'png','jpg','jpeg','gif','pdf'}` | **블랙리스트 금지** — 우회 무한대 |
| | **MIME 타입 서버 측 확인** | `python-magic`, `file` 명령, `mimetype` 라이브러리로 실제 타입 확인 | 클라이언트 MIME 위장 차단 |
| | **매직 바이트(파일 시그니처) 검증** | `PNG: 89 50 4E 47`, `JPEG: FF D8 FF`, `PDF: 25 50 44 46`, `GIF: 47 49 46 38` | **가장 확실한 내용 검증** |
| | **이미지 리사이징/재압축** | `PIL/Pillow`, `ImageMagick`, `GD`로 리사이즈 후 저장 | 임베디드 코드/메타데이터 제거, 매직 바이트 재생성 |
| | **EXIF/메타데이터 제거** | `exiftool -all=`, `PIL: img.save(..., exif=b'')` | 이미지 메타데이터 내 코드 제거 |
| **3. 저장/실행 환경 (필수)** | **업로드 디렉토리 웹 루트 밖 저장** | `/var/www/app/uploads/` → `/var/app_storage/uploads/` (DocumentRoot 밖) | **가장 확실** — 웹에서 직접 접근 불가 |
| | **업로드 디렉토리 실행 권한 제거** | Apache: `php_admin_flag engine off`, Nginx: `location ~ \.php$ { deny all; }` | **웹쉘 실행 원천 차단** |
| | **랜덤 파일명/UUID 사용** | `uuid4().hex + ext` → `a1b2c3d4.jpg` | 원본 파일명 경로 탐색/실행 방지 |
| | **확장자 강제 변경/고정** | 업로드 시 `.bin` 또는 `.dat`로 저장, 제공 시 원래 확장자 매핑 | 실행 확장자 원천 차단 |
| **4. 네트워크/인프라** | **WAF/CDN 규칙** | ModSecurity/Cloudflare/AWS WAF 업로드 규칙 | 알려진 악성 패턴/시그니처 차단 |
| | **바이러스/악성코드 스캔** | ClamAV, VirusTotal API, 상용 AV 엔진 연동 | 알려진 악성코드 차단 |
| **5. 모니터링/탐지** | **업로드 로깅/알림** | 파일명, 해시, 업로더 IP, 시점, MIME, 크기 기록 | 사후 분석/위협 헌팅 |
| | **웹쉘 탐지** | `webshell` 시그니처(`eval`, `system`, `exec`, `base64_decode`, `preg_replace` /e) 스캔 | 사후 탐지, 주기적 스캔 |

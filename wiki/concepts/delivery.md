# Delivery (전달) — 사이버 킬 체인 3단계

## Step 1: 단어 직역 및 쉬운 비유

### 1. 약자 풀이

**Delivery** = **Deliver** (전달하다, 배달하다) + **-y** (명사화 접미사)

| 영어 단어 | 직역 | 의미 |
|-----------|------|------|
| **Deliver** | 전달하다, 배달하다 | 물건을 목적지까지 옮기다 |
| **Delivery** | 전달, 배달, 인도 | 무언가를 목적지에 건네주는 행위 |

### 2. 의미 조합

> **"만들어진 무기(악성코드)를 대상 시스템/사용자에게 전달하는 과정 — 이메일, 웹, USB, 공급망 등 다양한 벡터를 통해 악성 파일이 피해자 환경에 도달하게 함"**

### 3. 강력한 비유: "폭탄을 담은 소포를 목표물 앞으로 배달하는 택배 기사"

```
┌────────────────────────────────────────────────────────────┐
│  상황: 테러리스트(공격자)가 조립한 폭탄(악성코드)을        │
│  목표 건물(대상 시스템/사용자) 문 앞까지 배달              │
└────────────────────────────────────────────────────────────┘

📦  **배달 경로(전달 벡터)** — 어떻게 폭탄을 문 앞까지 가져갈까?

  ① **우편물(피싱 이메일)**: "세금 환급 안내서" 위장 → 
      첨부파일(악성 매크로 문서) 열면 폭탄 작동

  ② **택배 상자(드라이브 바이 다운로드)**: 정상 사이트인 척 
      방문만 해도 자동 다운로드 → 사용자 몰래 폭탄 투하

  ③ **직접 전달(USB 드롭)**: 주차장에 "급여 명세서" 라벨 붙은 
      USB 놔둠 → 호기심에 꽂으면 폭탄 설치

  ④ **정기 배송(공급망 공격)**: 믿고 쓰는 소프트웨어 업데이트 서버 
      해킹 → 정상 업데이트에 폭탄 숨겨 배달

  ⑤ **내부 협력자(내부자 위협/자격증명 탈취)**: 
      정상 직원 계정 도용 → 사내 메신저/메일로 폭탄 전달

💡 **핵심 포인트**: 무기가 아무리 강력해도 **목표물 앞까지 배달 못 하면 무용지물**. 
공격자는 **"어떻게 하면 의심 없이 받게 할까?"** 에 집중합니다. 
사회공학(심리 조작)이 이 단계의 핵심 기술입니다.
```

## Step 2: 개념 시각화

![Delivery 비유 시각화: 택배 배달로 설명하는 악성코드 전달 — 공격자(해커), 악성 패키지(무기화된 멀웨어), 배달원(전달 벡터), 피싱 이메일(봉투), 악성 웹사이트(브라우저), USB 드라이브, 소프트웨어 업데이트, 대상 사용자(사무직) - 한글 레이블 포함](https://v3b.fal.media/files/b/0a9dfbf2/T8URX9DYhEagM4R_dmDjA_ujzQsM1q.png)

**이미지 설명**:
- **공격자(해커)** — 무기화된 악성코드를 전달하려는 주체
- **악성 패키지(무기화된 멀웨어)** — 익스플로잇 + 페이로드 + 드로퍼가 결합된 최종 무기
- **배달원(전달 벡터)** — 악성코드를 운반하는 매개체 (이메일, 웹, USB, 공급망 등)
- **피싱 이메일(봉투)** — 가장 흔한 전달 수단, 정상 메일로 위장
- **악성 웹사이트(브라우저)** — 드라이브 바이 다운로드, 워터링 홀 공격
- **USB 드라이브** — 물리적 전달, 소셜 엔지니어링 결합
- **소프트웨어 업데이트** — 공급망 공격, 신뢰 관계 악용
- **대상 사용자(사무직)** — 최종 수신자, 행동을 유도당함 (클릭, 열기, 실행)

## Step 3: 전문 용어 설명 (위키백과 기반)

### 전달 (Delivery, 사이버 킬 체인)

**정의**: **사이버 킬 체인의 3단계**로, 공격자가 2단계 무기화에서 제작한 악성코드를 **대상 시스템 또는 사용자 환경에 전달(전송·도달)시키는 과정**이다. 이 단계에서 공격자가 피해자와 **최초 접촉**을 하게 된다.

### 주요 전달 벡터 (Delivery Vectors)

| 벡터 분류 | 전달 방식 | 설명 | 대표 사례 |
|----------|-----------|------|-----------|
| **이메일 기반** | 피싱(Phishing) | 정상 기관/인물 위장 메일로 악성 첨부파일/링크 전송 | APT28, TA505, Business Email Compromise |
| | 스피어 피싱 | 특정 개인/조직 타겟팅, 맞춤형 내용 | APT1, Lazarus Group, FIN7 |
| | 웨일링 | 고위 임원(C-level) 타겟팅 | CEO 사기, 송금 사기 |
| **웹 기반** | 드라이브 바이 다운로드 | 정상 사이트 방문만으로 악성코드 자동 다운로드/실행 | Angler EK, RIG EK, Magnitude EK |
| | 워터링 홀 공격 | 대상 조직이 자주 방문하는 사이트 해킹 | APT19, APT32, DarkHotel |
| | 악성 광고(Malvertising) | 광고 네트워크 통해 악성코드 배포 | RoughTed, Kovter |
| **물리적** | USB 드롭/배드USB | 악성 펌웨어 심은 USB 기기 배포 | Stuxnet, Cottonmouth, Rubber Ducky |
| | 하드웨어 임플란트 | 서버/네트워크 장비 제조 단계 삽입 | NSA ANT 카탈로그, Supermicro 논란 |
| **공급망** | 소프트웨어 업데이트 해킹 | 정식 업데이트 채널에 악성코드 삽입 | SolarWinds SUNBURST, 3CX, CodeCov |
| | 오픈소스 의존성 오염 | npm/PyPI/Maven 패키지에 악성코드 | event-stream, ua-parser-js, log4j |
| **원격 접근** | 취약점 직접 익스플로잇 | 인터넷 노출 서비스 직접 공격 | EternalBlue, Log4Shell, ProxyShell |
| | 자격증명 재사용 | 탈취된 계정으로 VPN/RDP/SSH 접속 | 콜로니얼 파이프라인, 다크사이드 |
| **소셜 엔지니어링** | 보이싱/스미싱/프리텍스팅 | 전화/SMS/메신저로 심리 조작 | 택배 사칭, 관공서 사칭, 기술 지원 사칭 |

### 전달 단계에서의 공격자 고려사항

| 고려사항 | 설명 |
|----------|------|
| **전달 성공률** | 스팸 필터 우회, 샌드박스 회피, 사용자 클릭 유도(소셜 엔지니어링) |
| **추적 회피** | 익명 이메일, 토르/VPN, 도메인 프론팅, CDN 악용 |
| **타겟팅 정밀도** | 광범위 스프레이(Spray) vs 정밀 스피어(Spear) |
| **페이로드 크기/형식** | 이메일 첨부파일 크기 제한, 파일 형식 제한(EXE 차단 등) 회피 |
| **타이밍** | 업무 시간, 패치 주기, 휴일/연휴 악용 |

### 전달 단계 탐지 및 차단 (방어 관점)

| 방어 계층 | 대응 방안 |
|----------|-----------|
| **이메일 게이트웨이** | SPF/DKIM/DMARC 검증, 첨부파일 샌드박스 분석, URL 평판/리라이팅, 발신자 평판 |
| **엔드포인트** | EDR 파일/프로세스 모니터링, 매크로/스크립트 실행 제어(AMSI), 애플리케이션 화이트리스트 |
| **네트워크** | 웹 프록시/스위치 URL 필터링, DNS 싱크홀, TLS 검사, 악성 트래픽 시그니처(IDP/IPS) |
| **웹/애플리케이션** | WAF, CSP(Content Security Policy), SRI(Subresource Integrity), 취약점 패치 |
| **공급망** | SBOM(Software Bill of Materials), 서명 검증, 재현 가능 빌드, 의존성 스캐닝(SCA) |
| **인적/조직** | 피싱 시뮬레이션 교육, 보안 인식 제고, 의심 메일 신고 채널, MFA 강제 |

# Path Traversal (경로 순회, 디렉토리 순회) — 보안 용어 해설

## 참고 URL
- [ko.wikipedia.org](https://ko.wikipedia.org/wiki/경로_순회_공격)
- [ko.wikipedia.org](https://ko.wikipedia.org/wiki/OWASP)

## Step 1: 단어 직역 및 쉬운 비유

### 1. 용어 풀이

**Path Traversal** = **Path** (경로) + **Traversal** (순회, 횡단)

| 영어 단어 | 직역 | 의미 |
|-----------|------|------|
| **Path** | 경로, 길 | 파일/디렉토리 위치 표시 (`/home/user/file.txt`) |
| **Traversal** | 순회, 횡단, 지나가기 | 계층 구조를 따라 이동하며 탐색 |

**Directory Traversal** = **Directory** (디렉토리, 폴더) + **Traversal** (순회) = **폴더 구조를 벗어나 상위/타 폴더로 이동하는 공격**

### 2. 의미 조합

> **"사용자 입력(파일명, 경로 파라미터 등)에 `../` 같은 상위 디렉토리 이동 시퀀스가 포함되어, 의도한 디렉토리 밖의 파일/디렉토리에 접근/읽기/쓰기/실행하게 만드는 취약점"**

### 3. 강력한 비유: "열쇠 하나만 주었는데 마스터 키로 건물 전체 문 다 여는 손님"

```
┌────────────────────────────────────────────────────────────┐
│  상황: 호텔 프런트(서버)에서 고객(사용자)에게               │
│  "301호 열쇠(사용자 파일 접근 권한)"만 줬는데               │
│  고객이 마스터 키(`../`)로 101호, 201호, 관리자 사무실까지  │
│  다 열어서 들어감                                            │
└────────────────────────────────────────────────────────────┘

🔑  **마스터 키 시나리오 (Path Traversal 공격 흐름)**

  ① **정상 요청**: 사용자가 자기 프로필 사진 조회
     - 요청: `GET /profile?photo=myphoto.jpg`
     - 서버: `/var/www/uploads/myphoto.jpg` 읽어서 반환 ✓

  ② **공격자 악용**: 경로 파라미터에 `../` 주입
     - 요청: `GET /profile?photo=../../../etc/passwd`
     - 서버: `/var/www/uploads/../../../etc/passwd` 
           → `/etc/passwd`로 경로 해석 → **시스템 계정 파일 읽기**

  ③ **더 깊은 침투**: 
     - `../../../etc/shadow` → 비밀번호 해시 탈취
     - `../../../home/user/.ssh/id_rsa` → SSH 키 탈취 → 서버 접속
     - `../../../var/www/html/config.php` → DB 비밀번호/설정 탈취
     - `../../../proc/self/environ` → 환경변수(비밀키 등) 탈취

  ④ **쓰기/실행으로 확대 (Path Traversal + File Upload)**:
     - 업로드 파일명: `../../../var/www/html/shell.php`
     - 서버: 웹 루트에 웹쉘 저장 → **RCE 달성**

💡 **핵심 포인트**: 
- `../` (상위 디렉토리) `..` (현재 디렉토리) 조합으로 **디렉토리 트리 순회**
- **입력 검증/정규화 부족**이 근본 원인
- **읽기(LFI)** → **쓰기/실행(RCE)**로 확대 가능
- **윈도우**: `..\`, `..\/`, `..%5c`, `..%2f` 등 인코딩 변형
```

## Step 3: 전문 용어 설명 (위키백과/OWASP/PortSwigger 기반)
### Path Traversal (경로 순회, 디렉토리 순회)

**정의**: **Path Traversal(경로 순회, Directory Traversal)**은 애플리케이션이 **사용자 입력을 파일 경로로 사용할 때 `../`(상위 디렉토리)나 `..\`(윈도우) 같은 디렉토리 순회 시퀀스를 제대로 필터링/정규화하지 않아**, 공격자가 **의도된 디렉토리 트리 밖의 임의의 파일/디렉토리에 접근(읽기/쓰기/실행)할 수 있게 하는 취약점**이다. **Local File Inclusion(LFI)**과 밀접하게 관련된다.

### 공격 원리: 경로 정규화 실패

| 단계 | 설명 |
|------|------|
| **1. 사용자 입력 수용** | 파일명, 경로 파라미터, 쿠키, 헤더 등에서 경로 문자열 수용 |
| **2. 검증/정규화 부재** | `../`, `..\`, URL 인코딩, 이중 인코딩, 유니코드 변형 등 미필터링 |
| **3. 경로 결합** | 기본 디렉토리(`/var/www/uploads/`) + 사용자 입력(`../../../etc/passwd`) 결합 |
| **3. 경로 해석** | OS/파일시스템이 `../`를 상위 디렉토리로 해석 → 의도 밖 경로 접근 |
| **4. 파일 작업** | 읽기(정보 유출), 쓰기(웹쉘 심기), 실행(RCE), 삭제(파괴) 등 수행 |

### 주요 공격 벡터 및 페이로드

| 분류 | 페이로드 예시 | 설명 |
|------|---------------|------|
| **기본 상위 이동** | `../`, `..\`, `..../`, `....//` | 기본 상위 디렉토리 이동 |
| **URL 인코딩** | `%2e%2e%2f`, `%2e%2e/`, `%2e%2e%5c` | `/` → `%2f`, `\` → `%5c`, `.` → `%2e` |
| **이중 인코딩** | `%252e%252e%252f` | `%` → `%25` → 이중 디코딩 시 `../` 복원 |
| **유니코드/과긴 UTF-8** | `..%c0%af`, `..%c1%9c` | UTF-8 오버롱 시퀀스로 `/`/`\` 표현 |
| **NULL 바이트** | `../../../etc/passwd%00` | C 문자열 종료로 확장자 검증 우회 (구버전) |
| **경로 혼합** | `..;/`, `.././`, `..\./` | 파서별 차이 악용 |
| **윈도우 특화** | `..\`, `..%5c`, `..%c1%1c`, `..%ff%5c` | 백슬래시, 드라이브 레터(`C:`), UNC 경로(`\\server\share`) |
| **절대 경로** | `/etc/passwd`, `C:\Windows\system32\drivers\etc\hosts` | 상대 경로 제한 우회 |
| **심볼릭 링크** | `upload/` → `/etc/` 심볼릭 링크 생성 후 접근 | 심볼릭 링크 따라가기 |

### 공격 유형별 영향

| 공격 유형 | 설명 | 영향 |
|----------|------|------|
| **파일 읽기 (LFI)** | `/etc/passwd`, `/etc/shadow`, 설정파일, 소스코드, 로그, SSH 키 | 정보 유출, 자격증명 탈취, 추가 공격 기점 |
| **파일 쓰기** | 웹쉘 업로드(`../../../var/www/html/shell.php`), 설정파일 변조 | RCE, 영구 지속성, 변조 |
| **파일 실행** | 업로드된 스크립트 실행, 기존 바이너리 실행 | RCE, 권한 상승 |
| **파일 삭제** | `../../../var/www/html/index.php` 삭제 | 서비스 중단, 증거 인멸 |
| **소스코드 유출** | `../../../var/www/html/index.php`, `.git/config` | 로직 분석, 추가 취약점 발견, 하드코딩 비밀키 |

### 윈도우 vs 리눅스 경로 차이

| 구분 | 리눅스/유닉스 | 윈도우 |
|------|---------------|--------|
| **구분자** | `/` (포워드 슬래시) | `\` (백슬래시) 또는 `/` (둘 다 지원) |
| **상위 디렉토리** | `../` | `..\` |
| **루트** | `/` | `C:\`, `D:\` 등 드라이브 레터 |
| **현재 디렉토리** | `./` | `.\` |
| **UNC 경로** | 지원 안 함 | `\\server\share\path` |
| **대소문자** | 구분함 (Case-sensitive) | 구분 안 함 (Case-insensitive) |
| **예약어** | 없음 | `CON`, `PRN`, `AUX`, `NUL`, `COM1~9`, `LPT1~9` |

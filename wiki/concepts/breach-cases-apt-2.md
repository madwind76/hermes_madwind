# 실제 침해 사례 — APT / 데이터 유출 (Part 2)

> [[breach-cases-apt]]의 분할 페이지입니다.

### 12. Equifax (2017) ⭐⭐⭐
| 항목 | 내용 |
|------|------|
| **공격 기법** | CVE-2017-5638 — Apache Struts2 OGNL 인젝션 |
| **실습 도구** | Vulhub `struts2/CVE-2017-5638`, struts2-rest-showcase |
| **실습 내용** | Struts2 취약점 스캔 → OGNL 페이로드 전송 → RCE → 패치 미적용 로그 분석 |
| **교육 포인트** | CVSS 점수와 실제 위험의 차이, 패치 지연의 대가, 1.43억 건 데이터 유출 |

#### 사고 상세
| 세부 항목 | 내용 |
|----------|------|
| **발생일** | 2017-05-13 (최초 침투) |
| **발견일** | 2017-07-29 (내부 보안팀) |
| **공격 타임라인** | 2017-03: Apache Struts CVE-2017-5638 패치 배포 → Equifax가 2개월간 패치 미적용 → 2017-05-13: 해커 침투 성공 → 7월 29일까지 데이터 지속 유출 → 2017-09-07: Equifax 공식 발표 → CEO 즉시 퇴임, 7억 달러 FTC 합의 |
| **피해 규모** | **1.47억 명 개인정보** (SSN, 생년월일, 주소, 운전면허 번호). 주가 30% 폭락. FTC 합의 7억 달러. 총 비용 14억 달러 이상 |
| **발견 경로** | Equifax 보안팀이 내부 네트워크에서 비정상 트래픽 발견 → 포렌식 |
| **배후** | 중국 해커 그룹 (2017년 말 FBI 수사 중 중국인 특정). 2019년 연방 검찰 기소 |

### 13. Capital One AWS SSRF (2019) ⭐⭐⭐
| 항목 | 내용 |
|------|------|
| **공격 기법** | WAF SSRF 우회 → EC2 메타데이터 API → IAM 역할 권한 남용 |
| **실습 도구** | Localstack + 취약 Flask 앱, Terraform으로 AWS 환경 시뮬레이션 |
| **실습 내용** | SSRF 탐지 → 메타데이터 서비스 접근 → IAM 자격증명 탈취 → S3 데이터 유출 |
| **교육 포인트** | 클라우드 공유 책임 모델, IMDSv1 vs IMDSv2, IAM 최소 권한 |

#### 사고 상세
| 세부 항목 | 내용 |
|----------|------|
| **발생일** | 2019-03-12 ~ 2019-07-17 |
| **발견일** | 2019-07-17 (외부 연구원 신고) |
| **공격 타임라인** | 2019-03-12: Paige Thompson(전 AWS 직원)이 Capital One WAF 설정 오차(SSRF) 발견 → EC2 메타데이터 API로 IAM 역할 자격증명 탈취 → S3 버킷 700개 데이터 접근 → GitHub에 자신의 공격 자랑글 게시 → 누군가 신고 → 2019-07-29: FBI 체포 |
| **피해 규모** | **1.06억 명 신용카드 신청 데이터** (이름, SSN, 운전면허, 소득). 8만 명 은행 계좌 번호 유출. Capital One 1.5억 달러 지출 (소송·합의·개선) |
| **발견 경로** | GitHub 사용자가 Paige Thompson의 게시물 \"I've exfiltrated Capital One data\" 발견 → Capital One 신고 |
| **배후** | Paige Thompson (개인 해커, 전 AWS 직원). 2022년 유죄 인정, 징역 20년 선고 |

### 14. Marriott / Starwood (2018) ⭐⭐
| 항목 | 내용 |
|------|------|
| **공격 기법** | SQL 인젝션 → 백도어 설치 → 4년간 데이터 유출 |
| **실습 도구** | SQLi-lab, 지속적 백도어 시뮬레이션 (cron job 등) |
| **실습 내용** | SQL 인젝션 → 장기 침투 탐지 (DB 감사 로그, 크론 작업 이상) |
| **교육 포인트** | 장기 침투 탐지의 어려움, 암호화 저장의 중요성 |

#### 사고 상세
| 세부 항목 | 내용 |
|----------|------|
| **발생일** | 2014년경 ~ 2018-09 |
| **발견일** | 2018-09-08 (내부 보안 경보) |
| **공격 타임라인** | 2014년경: Starwood 호텔 예약 시스템 해킹 → 악성코드 심어짐 → **4년간 무증상 데이터 유출** → 2015년: Marriott가 Starwood 인수 (취약점 승계) → 2018-09: Marriott 보안팀이 Starwood DB에서 이상 쿼리 발견 → 2018-11-30: Marriott 공개 |
| **피해 규모** | **5억 명 고객 기록** (이름, 주소, 전화번호, 이메일, 여권번호). 영국 ICO 1.84억 달러 벌금. FTC 합의 5,200만 달러 |
| **발견 경로** | Marriott 내부 보안 시스템이 Starwood 데이터베이스 비정상 접근 기록 탐지 |
| **배후** | 중국 해커 그룹 (APT10?). 2019년 중국 안전부 소속 세 명 특정 |

### 15. Uber (2022) ⭐⭐
| 항목 | 내용 |
|------|------|
| **공격 기법** | 소셜 엔지니어링 + MFA 피로 공격 → 계약자 계정 탈취 |
| **실습 도구** | DUO 유사 시뮬레이션, 피싱 시뮬레이션 플랫폼 |
| **실습 내용** | MFA 푸시 피로 공격 시뮬레이션, VPN 로그 분석, ID 보호 |
| **교육 포인트** | MFA도 완벽하지 않음, 소셜 엔지니어링 대응, 계약자 계정 관리 |

#### 사고 상세
| 세부 항목 | 내용 |
|----------|------|
| **발생일** | 2022-09-15 |
| **발견일** | 2022-09-15 (당일) |
| **공격 타임라인** | LAPSUS$ 18세 해커가 Uber 직원 Slack 메시지에서 VPN 크리덴셜 획득 → MFA 피로 공격(MFA Fatigue, 2FA Push 수십 회 전송) → HackerOne 계정 접근 → Slack/AWS/G-Suite/코드 저장소 광범위 침투 → Slack에 \"I am a hacker and Uber has suffered a data breach\" 문구 게시 |
| **피해 규모** | 내부 시스템, DB, 코드 저장소, HackerOne 계정 접근. **57백만 명 데이터 접근 가능성**. 주가 5% 하락 |
| **발견 경로** | 공격자가 Uber HackerOne 페이지에 직접 메시지 → HackerOne 직원 인지 → Uber 보안팀 연락 |
| **배후** | LAPSUS$ (18세 영국 청소년, 이후 영국 법원 처벌) |

### 16. GitLab (CVE-2023-7028) (2023) ⭐⭐⭐
| 항목 | 내용 |
|------|------|
| **공격 기법** | CVE-2023-7028 — 패스워드 재설정 이메일 스푸핑 → 계정 탈취 |
| **실습 도구** | 공식 GitLab Docker 취약 버전 |
| **실습 내용** | 비즈니스 로직 결함 분석, 이메일 헤더 검증 우회, 권한 상승 |
| **교육 포인트** | 비즈니스 로직 취약점, 이메일 인증의 취약성 |

#### 사고 상세
| 세부 항목 | 내용 |
|----------|------|
| **발생일** | 2023-12 (발견) |
| **발견일** | 2024-01-10 (GitLab 보안팀) |
| **공격 타임라인** | GitLab CE/EE 취약점: 비밀번호 리셋 시 이메일 인증 없이 임의 이메일로 리셋 링크 전송 가능 → 계정 탈취 가능 → 2024-01-11: GitLab 패치 발표 (5명 이하 자체관리 라이선스 영향) |
| **피해 규모** | 제한적 (자체관리 GitLab). 공식 피해 사례 미보고 |
| **발견 경로** | GitLab 보안팀 내부 감사 중 발견 |
| **배후** | 취약점 자체. 특정 해커 그룹 아님 |

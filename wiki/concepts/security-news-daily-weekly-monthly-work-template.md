# Security News Daily Weekly Monthly Work Template

이 문서는 보안 뉴스 수집 작업을 **매일/매주/매월** 반복할 수 있도록 나눈 작업 템플릿입니다.  
실제로는 이 템플릿을 따라 움직이면, 수집부터 발행까지 흐름이 끊기지 않습니다.

## 1. 매일 작업

### 목표
- 새로운 뉴스와 공지의 누락을 막습니다.
- 오늘 들어온 사건 후보를 빠르게 적재합니다.

### 작업 항목
- [ ] RSS/공식공지/벤더 블로그를 확인합니다.
- [ ] 새 기사 제목과 URL을 수집합니다.
- [ ] 중복 기사 후보를 먼저 표시합니다.
- [ ] 야생 공격, KEV, 긴급 패치, 유출 사건을 우선 체크합니다.
- [ ] 사건군 후보 메모를 간단히 남깁니다.

### 산출물
- 일일 수집 목록
- 사건군 후보 메모
- 긴급 항목 우선순위 리스트

## 2. 매주 작업

### 목표
- 주간 보고서와 뉴스레터를 만들 수 있을 만큼 사건군을 정리합니다.

### 작업 항목
- [ ] 한 주 동안 수집된 항목을 사건군으로 묶습니다.
- [ ] 대표 출처와 보조 출처를 정리합니다.
- [ ] `p1 / p2 / p3` 우선순위를 확정합니다.
- [ ] 이번 주 핵심 3~5개를 고릅니다.
- [ ] 각 기사에 2~3줄 한글 요약을 붙입니다.
- [ ] 주간 뉴스레터 초안을 만듭니다.
- [ ] 발행 체크리스트로 최종 점검합니다.

### 산출물
- 주간 사건군 테이블
- 주간 보고서 초안
- 주간 뉴스레터 초안
- 발행 체크리스트 완료 기록

## 3. 매월 작업

### 목표
- 한 달 동안 반복된 테마를 분석하고, 월간 보고서와 월간 뉴스레터를 만듭니다.

### 작업 항목
- [ ] 같은 사건군이 여러 주에 걸쳐 반복됐는지 확인합니다.
- [ ] 벤더별/제품군별/지역별 반복 패턴을 묶습니다.
- [ ] 전월 대비 변화 포인트를 정리합니다.
- [ ] 월간 보고서용 핵심 테마를 3~5개로 압축합니다.
- [ ] 월간 뉴스레터용 대표 기사와 요약을 정리합니다.
- [ ] 다음 달 관찰 포인트를 적습니다.

### 산출물
- 월간 테마 클러스터
- 월간 보고서 초안
- 월간 뉴스레터 초안
- 다음 달 관찰 포인트 목록

## 4. 반복 운영 규칙

- 일간은 **누락 방지**가 목적입니다.
- 주간은 **편집과 발행**이 목적입니다.
- 월간은 **추세 해석**이 목적입니다.
- 같은 사건은 기사보다 **사건군**으로 관리합니다.
- 뉴스레터는 보고서와 분리해 읽기 쉽게 편집합니다.

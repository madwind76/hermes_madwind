# IDS (Intrusion Detection System) — 침입 탐지 시스템

## Step 1: 단어 직역 및 쉬운 비유

### 1. 약자 풀이
**IDS** = **I**ntrusion **D**etection **S**ystem

| 약자 | 영어 단어 | 직역 | 의미 |
|------|-----------|------|------|
| **I** | **Intrusion** | 침입, 침범, 무단 침투 | 허가 없이 내부로 들어오는 행위 |
| **D** | **Detection** | 탐지, 발견, 감지 | 무언가를 찾아내거나 알아채는 행위 |
| **S** | **System** | 체계, 시스템 | 여러 구성 요소가 유기적으로 작동하는 장치/프로그램 |

### 2. 의미 조합
> **네트워크나 시스템에서 발생하는 악의적 활동이나 보안 정책 위반을 실시간으로 모니터링하여 탐지·분석·경고하는 보안 시스템 — 하지만 직접 차단하지는 않고 관리자에게 알린다**

### 3. 강력한 비유: "학교 경비실에서 모니터 보는 경비원"

| 비유 요소 | 대응 개념 | 설명 |
|-----------|-----------|------|
| **학교 건물** | **네트워크/시스템** | 보호 대상 인프라 |
| **학교 정문** | **네트워크 게이트웨이, 스위치 포트** | 트래픽이 지나가는 지점 |
| **경비실(학교 앞)** | **IDS 센서(Network TAP / Port Mirroring)** | 모든 방문객(트래픽)이 지나가는 곳에 설치 |
| **경비원 모니터** | **패킷 분석 엔진** | 트래픽 내용을 실시간 분석 |
| **수배 전단지(범죄자 사진)** | **시그니처(Signature)** | 알려진 공격 패턴 데이터베이스 |
| **"수상한 사람이야!" 보고** | **이상 행위 탐지(Anomaly) 경고** | 평소와 다른 패턴 감지 시 경고 |
| **방문 명부 기록** | **로그 저장 (Syslog, PCAP)** | 모든 이벤트 기록 저장 |
| **경비원이 직접 문 잠그지 않음** | **IPS와의 차이점: 차단 기능 없음** | IDS는 감지만 하고 차단 기능은 없음 |
| **"삐삐삐" 경보만 울림** | **알림(Alert)만 발생** | 차단은 별도 장비(방화벽/IPS)가 수행 |

**동작 순서**:
1. 방문객(트래픽)이 학교 정문 통과 → 모든 패킷이 IDS 센서를 지나감
2. 경비원(IDS 엔진)이 방문객 행동 관찰 → 패킷 내용 분석 (프로토콜, 페이로드, 헤더)
3. 수배 전단지와 대조 → 시그니처 기반 탐지 (Snort/Suricata 룰 매칭)
4. "평소와 다른 행동이다" 감지 → 이상 행위 탐지 (ML 모델, 통계적 이상치)
5. 경비원이 "수상한 사람 발견!" 전화 → 알림/경고 (Alert) → SOC 담당자 확인
6. 방문 명부에 기록 → 로그 저장 (이벤트 타임스탬프, IP, 프로토콜, 페이로드)

**핵심 포인트**: **IDS는 감시 카메라(CCTV)와 같습니다** — 도둑이 침입하는 장면을 포착해서 경비실에 알람을 울리지만, 직접 문을 잠그거나 도둑을 막지는 않습니다. "누가, 언제, 어떻게 들어왔는지"를 기록하는 게 주 임무입니다. **직접 차단하려면 IPS(Intrusion Prevention System)**가 필요합니다.

**이미지 설명**:
- **네트워크(학교)** — 보호 대상 인프라
- **IDS 센서(경비실)** — 네트워크 TAP/포트 미러링으로 트래픽 수집
- **트래픽 모니터링(경비원 모니터)** — 실시간 패킷 분석
- **정상 트래픽(일반 방문객 - 초록)** — 통과 허용, 로그만 기록
- **악성 트래픽(해커 - 빨강 경고)** — 시그니처/이상치 매칭 → 경고 발생
- **시그니처 기반 탐지(수배 전단지)** — 알려진 공격 패턴 DB(Snort/Suricata 룰)
- **이상 행위 탐지(이상한 행동 감지)** — ML/통계 기반 평소와 다른 패턴
- **경고 발생(알람) — 차단은 안 함** — IDS vs IPS 핵심 차이

## Step 3: 전문 용어 설명 (Wikipedia 기반)

### 침입 탐지 시스템 (Intrusion Detection System, IDS)

**정의**: 네트워크 트래픽 또는 시스템 활동을 **실시간으로 모니터링**하여 악의적인 활동이나 보안 정책 위반을 **탐지·분석·기록·경고**하는 보안 시스템 또는 소프트웨어이다. IDS는 **탐지만 수행하고 능동적 차단은 하지 않는 점**에서 IPS(Intrusion Prevention System)와 구분된다.

---

### IDS 분류 체계

#### 1. 탐지 위치에 따른 분류

| 유형 | 설명 | 장점 | 단점 | 대표 도구 |
|------|------|------|------|-----------|
| **NIDS (Network-based)** | 네트워크 세그먼트의 트래픽 전체를 수집·분석 | 호스트 무관, 광범위 탐지, OS 독립 | 암호화 트래픽 분석 불가, 대규모 네트워크에서 병목 | Snort, Suricata, Zeek (BRO) |
| **HIDS (Host-based)** | 개별 호스트 내부의 로그·프로세스·파일·레지스트리 모니터링 | 시스템 콜 수준 탐지, 암호화 트래픽 내부 분석 가능, 파일 무결성 | 호스트별 에이전트 필요, 관리 복잡도 증가 | OSSEC, Wazuh, AIDE, Tripwire, Osquery |
| **DIDS (Distributed)** | 여러 IDS 센서를 분산 배치, 중앙 콘솔에서 통합 관리 | 대규모 네트워크 커버리지 | 중앙 분석 서버 SPOF 가능성 | Snort + Barnyard2 + PulledPork |

#### 2. 탐지 방법에 따른 분류

| 탐지 방식 | 설명 | 장점 | 단점 |
|-----------|------|------|------|
| **시그니처 기반 (Signature-based)** | 알려진 공격 패턴 DB와 트래픽 정확 매칭 | 오탐률 낮음, 즉시 탐지, 증거력 높음 | 변종/제로데이 공격 미탐지, 시그니처 업데이트 필요 |
| **이상 행위 기반 (Anomaly-based)** | 정상 프로파일 학습 → 통계적/ML 기반 이상치 탐지 | 제로데이/변종 탐지 가능, 시그니처 불필요 | 오탐률 높음, 트레이닝 필요, 적응형 공격에 취약 |
| **상태 프로토콜 분석 (Stateful Analysis)** | RFC 기반 프로토콜 상태 머신 기준 비정상 동작 탐지 | 프로토콜 위반 정확 탐지 | 프로토콜 종속적, 커스텀 프로토콜 미지원 |
| **ML/AI 기반 탐지** | 머신러닝/딥러닝 모델이 트래픽 특징 학습 후 분류 | 고도화된 변종 탐지, 자동 특징 추출 | 데이터 의존성, 블랙박스 의사결정, 허위 양성 |

### IDS 배치 아키텍처

```
┌─────────────┐
│   인터넷     │
└──────┬──────┘
       │
┌──────┴──────┐
│   방화벽     │
└──────┬──────┘
       │
┌──────┴────────────┐
│ IDS 센서 (Port Mirror)│ ← 트래픽 복사본 (인라인 아님)
└──────┬────────────┘
       │
┌──────┴──────┐
│ 내부 네트워크 │
└─────────────┘
       ▲
       │
┌──────┴─────────┐
│ SIEM / SOC 분석가 │
└────────────────┘
```

> **핵심**: IDS 센서는 인라인이 아닌 포트 미러링(SPAN/TAP) 방식 → IDS 장애 시 네트워크 영향 없음

### IDS vs IPS vs 방화벽 비교

| 구분 | 방화벽 (Firewall) | IDS | IPS |
|------|------------------|-----|-----|
| **위치** | 네트워크 경계 (인라인) | 포트 미러링 (오프라인) | 네트워크 경로 (인라인) |
| **동작** | ACL 기반 차단/허용 | 탐지 + 경고 | 탐지 + **실시간 차단** |
| **프로토콜** | L3-L4 (IP/포트) | L3-L7 (페이로드 분석) | L3-L7 전체 |
| **장애 영향** | 네트워크 단절 | 무영향 (passive) | 네트워크 단절 가능 |
| **지연 시간** | 약간 증가 | 거의 없음 (복사본) | 증가 (인라인 처리) |

### 주요 IDS 엔진 비교

| 엔진 | 언어 | 처리량 | 멀티스레드 | 특징 |
|------|------|--------|-----------|------|
| **Snort** | C | ~1Gbps | 제한적 | 가장 오래된 표준 (1998), 풍부한 룰셋(ET, VRT) |
| **Suricata** | C/Rust | **~10Gbps+** | ✅ 완벽 | 차세대 표준, GPU 가속(CUDA), TLS/HTTP2/SSH 파싱 |
| **Zeek (BRO)** | C++/Script | ~5Gbps | ✅ | 트래픽 분석 프레임워크, 강력한 로깅, 커스텀 분석 스크립트 |
| **OSSEC/Wazuh** | C/Python | N/A | ✅ | HIDS 전용 — 로그 분석, 파일 무결성, 루트킷 탐지, SIEM 통합 |

### IDS 탐지 우회 기법 (Evasion) 및 대응

| 우회 기법 | 설명 | IDS 대응 |
|-----------|------|----------|
| **프래그먼트 공격** | 공격 페이로드를 IP 프래그먼트로 분할 | IP 재조립 엔진 (Snort frag3, Suricata defrag) |
| **인코딩/난독화** | URL 인코딩, Base64 등으로 시그니처 회피 | 정규화 전처리기 (Snort http_inspect) |
| **다형성 페이로드** | 매 실행 다른 형태 쉘코드 | 행위 기반/이상 탐지, 에뮬레이션 |
| **암호화 트래픽 (TLS)** | HTTPS로 공격 전송, 페이로드 분석 불가 | TLS 프록시/복호화, JA3 핑거프린팅 |
| **타이밍 공격** | 매우 느린 속도로 공격 (Slowloris) | 세션 타임아웃/상태 추적, 비율 기반 탐지 |

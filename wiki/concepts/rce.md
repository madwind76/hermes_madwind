# RCE (Remote Code Execution) — 원격 코드 실행

## Step 1: 단어 직역 및 쉬운 비유

### 1. 약자 풀이
**RCE** = **Remote Code Execution**

| 영어 단어 | 직역 | 의미 |
|-----------|------|------|
| **Remote** | 원격의, 먼 곳의 | 물리적으로 떨어져 있는 위치에서 |
| **Code** | 코드, 프로그램 코드 | 컴퓨터가 실행할 수 있는 명령어 집합 |
| **Execution** | 실행, 수행 | 프로그램이 실제로 작동하게 만드는 행위 |

### 2. 의미 조합
> **공격자가 원격(떨어진 위치)에서 대상 시스템에 침투해, 자신의 의도대로 임의의 코드(프로그램)를 실행시킬 수 있는 치명적인 취약점**

### 3. 강력한 비유: "열린 창문으로 들어온 도둑이 내 집에서 주인 행세하기"

- **당신의 집 = 서버(타겟 시스템)**
- **열린 창문/문 = 취약점(보안 구멍)**
- **도둑 = 원격 공격자(해커)**
- **도둑이 가져온 요리 도구 = 악성 코드(페이로드)**
- **도둑이 당신의 주방에서 마음대로 요리함 = 임의 코드 실행**
- **도둑이 TV 리모컨으로 채널 돌림 = 시스템 완전 제어권 탈취**
- **도둑이 안방 금고 열어서 현금 훔쳐감 = 중요 데이터 탈취**

**핵심 포인트**: 주인이 집에 없어도, 열쇠가 없어도, **그저 창문이 열려 있다는 이유만으로** 남이 내 집 안에서 **하고 싶은 거 다 하고 나갈 수 있는 상황**입니다. 이게 바로 RCE입니다.

## Step 3: 전문 용어 설명 (Wikipedia 기반)

### 원격 코드 실행(Remote Code Execution, RCE)

**정의**: 공격자가 **물리적 접근 없이 네트워크를 통해** 대상 시스템에서 **임의의 명령어 또는 코드를 실행**할 수 있게 하는 보안 취약점의 클래스이다.

### 작동 원리

1. **취약점 유발**: 대상 애플리케이션/서비스의 입력 검증 부재, 메모리 관리 오류(버퍼 오버플로우, Use-After-Free), 안전하지 않은 역직렬화, 템플릿 인젝션, 명령어 인젝션 등으로 공격자 제어 데이터가 코드 실행 경로에 도달
2. **익스플로잇 전달**: 악의적 페이로드(쉘코드, 스크립트, 직렬화 객체 등)를 네트워크 패킷, HTTP 요청, 파일 업로드 등으로 전송
3. **코드 실행**: 취약한 컴포넌트가 공격자 데이터를 코드로 해석·실행 → 공격자 권한으로 프로세스 생성
4. **포스트 익스플로잇**: 리버스 쉘 획득, 권한 상승(PrivEsc), 횡적 이동(Lateral Movement), 데이터 유출, 지속성 확보

### 주요 특징

| 특징 | 설명 |
|------|------|
| **심각도** | **CVSS 9.0~10.0 (Critical)** — 인증 없이 원격에서 완전 제어 가능 |
| **영향 범위** | 서버 완전 장악 → 내부망 침투 기점 → 랜섬웨어 배포, 데이터 탈취, 암호화폐 채굴 등 |
| **대표 사례** | Log4Shell(CVE-2021-44228), EternalBlue(CVE-2017-0144), ProxyShell, Struts2 RCE, Spring4Shell |
| **공격 벡터** | 네트워크 서비스(HTTP, SMB, RDP, SSH), 웹 애플리케이션, 미들웨어, 프레임워크 |

### 방어 대책

- **입력 검증 & 새니타이징**: 모든 사용자 입력 검증, 허용 목록(allowlist) 기반 필터링
- **최소 권한 원칙**: 서비스 계정 권한 최소화, 컨테이너/샌드박스 격리
- **패치 관리**: 공개된 CVE 신속 패치, WAF/IPS 시그니처 업데이트
- **메모리 보호**: ASLR, DEP/NX, Stack Canary, CFI 등 컴파일러/OS 레벨 보호 활성화
- **모니터링**: 비정상 프로세스 실행, 네트워크 연결, 파일 변조 탐지(EDR, SIEM)
